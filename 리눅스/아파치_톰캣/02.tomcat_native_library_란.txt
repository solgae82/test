Tomcat Native Library(libtcnative)는 
Tomcat이 APR(Apache Portable Runtime)을 사용하여 
성능을 극대화할 수 있게 해주는 
JNI(Java Native Interface) 기반의 C/C++ 라이브러리입니다

(핵심 특징과 역할)
	
	- APR 기반 Native Connector: 
		Java의 Blocking I/O 대신 운영체제(OS)의 네이티브 기능을 직접 호출하여 
		고성능을 제공하는 APR 기반 커넥터(Http11AprProtocol)를 사용하게 합니다.
		
	- JNI Wrapper: 
		자바 환경에서 C/C++로 작성된 APR 및 OpenSSL 라이브러리에 
		접근할 수 있도록 도와주는 역할을 합니다.
		
	- 성능 향상: 
		자바 가상 머신(JVM) 기반의 기본 I/O 대신 
		운영체제의 네이티브 소켓 및 I/O를 사용하여 처리 속도를 높입니다.
		특히 높은 트래픽 환경에서 정적 파일 처리 및 SSL 연결 속도를 크게 개선합니다

	- 고효율 SSL/TLS: 
		OpenSSL을 활용하여 자바 JSSE 방식보다 
		훨씬 빠른 암호화/복호화 성능을 제공합니다.

	- Non-blocking I/O: 
		Keep-Alive 요청 등에서 더 효율적인 리소스 관리가 가능합니다.

설치 및 적용

	필수 여부: 
		필수는 아니며, 높은 트래픽을 처리하는 
		운영 환경(Production)에서 주로 권장됩니다.

	확인 방법: 
		톰캣 구동 로그에서 
		'The APR based Apache Tomcat Native library was not found' 
		메시지가 보인다면 해당 라이브러리가 로드되지 않은 것입니다.
	

------------------- 요약

- Tomcat Native Library는 고성능 APR(Apache Portable Runtime)을 사용하기 위한 
JNI(java native interface) 기반의 C/C++ 라이브러리.

- java 프로그램내에서 네이티브(.so) 모듈 기능을 사용하기 위해 
JNI 기반의 (Tomcat Native Library)로 로드해서 사용한다
