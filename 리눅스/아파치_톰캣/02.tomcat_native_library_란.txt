톰캣 네이티브 라이브러리(Tomcat Native Library)는 
아파치 톰캣(Apache Tomcat)의 성능과 호환성을 향상시키기 위해 선택적으로 사용하는 
APR(Apache Portable Runtime) 기반의 C 언어 라이브러리입니다. 

Java의 I/O 및 SSL 처리 방식을 네이티브 코드로 대체하여 고성능 네트워크 네트워킹과 
보안 통신(TLS/SSL)을 제공합니다.

핵심 특징 및 장점

	- 성능 향상: 
		자바 가상 머신(JVM) 기반의 기본 I/O 대신 
		운영체제의 네이티브 소켓 및 I/O를 사용하여 처리 속도를 높입니다.

	- 고효율 SSL/TLS: 
		OpenSSL을 활용하여 자바 JSSE 방식보다 
		훨씬 빠른 암호화/복호화 성능을 제공합니다.

	- Non-blocking I/O: 
		Keep-Alive 요청 등에서 더 효율적인 리소스 관리가 가능합니다.

주요 구성 요소 (APR)
	톰캣 네이티브 라이브러리는 APR(Apache Portable Runtime)을 사용하여 
	다음 기능들을 제공합니다

	- Socket Connector: 최적화된 네트워크 연결 처리.
	- Random Number Generator: 보안 강화.

설치 및 적용

	필수 여부: 
		필수는 아니며, 높은 트래픽을 처리하는 
		운영 환경(Production)에서 주로 권장됩니다.

	확인 방법: 
		톰캣 구동 로그에서 
		'The APR based Apache Tomcat Native library was not found' 
		메시지가 보인다면 해당 라이브러리가 로드되지 않은 것입니다.
	
	
즉, 톰캣 네이티브 라이브러리는 

"자바 환경에서 Apache 웹 서버 수준의 고성능 I/O와 안정성을 얻기 위한 보조 라이브러리" 

라고 이해하면 됩니다

- 톰캣 네이티브 라이브러리는 APR(Apache Portable Runtime)을 사용한다.
