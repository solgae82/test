<아파치 톰캣 연동을 위한 mod_jk 설치>

아파치 홈 : /usr/local/apache2.4/bin
톰캣 홈 : /usr/local/apache-tomcat-9.0.113

1. 소스 컴파일 설치
	cd /usr/local/src

	tar xvfz tomcat-connectors-1.2.50-src.tar.gz

	cd ./tomcat-connectors-1.2.50-src/native

	#아파치 홈/bin에 있는 apxs 를 지정
	./configure --with-apxs=/usr/local/apache2.4/bin/apxs
	 
	#컴파일
	make
	 
	#설치
	make install

	#확인 (mod_jk 생성되었는지)
	cd /usr/local/apache2.4/modules
	ls | grep mod_jk
	mod_jk.so #생성 확인


2. workers.propeties 파일 정의

	cd /usr/local/apache2.4/conf
	vi workers.propeties
	
	worker.list=tomcat01
	worker.tomcat01.type=ajp13
	worker.tomcat01.host=127.0.0.1	 #이거 중요
	worker.tomcat01.port=8009
	worker.tomcat01.secret=1234
	
3 run 디렉토리 만들기
	cd /usr/local/apache2.4/
	mkdir run
	
4. httpd.conf 설정
	
	# httpd.conf 추가 편집
	cd /usr/local/apache2.4/conf
	vi httpd.conf
	..	
	
	# httpd.conf 하단에 아래 내용을 추가합니다.

	# mod_jk.so 모듈의 위치
	LoadModule jk_module modules/mod_jk.so

	# workers.properties 설정
	JkWorkersFile conf/workers.properties

	# mod_jk log 위치
	JkLogFile logs/mod_jk.log

	# Load balancing worekrs will not function properly 오류 대응
	JkShmFile run/mod_jk.shm

	# /*.jsp 요청을 모두 tomcat01으로 전송
	JkMount /*.jsp tomcat01

	
5. 설정 후 아파치 시작 (테스트)
	cd /usr/local/apache2.4/bin
	./apachectl start

	 #아파치 홈 run 디렉토리에 
		mod_jk.shm.6803  mod_jk.shm.6803.lock
	파일이 생성되는지 확인해본다

============================= Tomcat Native Library 설치

	(톰캣 설치시 Tomcat Native Library 까지 설치 안 되었다면 설치한다)
	
	1. Tomcat Native Library 컴파일 설치  
				
		cd /usr/local/tomcat-9.0.113/bin 
		tar xvfz tomcat-native.tar.gz
		ls 
			tomcat-native-1.3.1-src # .gz 압축해제 디렉토리 확인
			
		cd tomcat-native-1.3.1-src/native

		#설정 옵션을 자세히 보자 (검색해서 파악하면 좋다)
		./configure --with-apr=/usr/local/apr-1.7.6/bin/apr-1-config \
		--with-java-home=/usr/local/jdk-11.0.0.2 \
		--with-ssl=yes

		# 컴파일 및 설치
		make && make install

	2. Tomcat Native Library  arp/lib 설치 되었는지 확인 
		:'libtcnative*' 라이브러리 파일들 생성 확인

		ls /usr/local/apr-1.7.6/lib
		
			apr.exp      libapr-1.so        libtcnative-1.a   libtcnative-1.so.0
			libapr-1.a   libapr-1.so.0      libtcnative-1.la  libtcnative-1.so.0.3.1
			libapr-1.la  libapr-1.so.0.7.6  libtcnative-1.so  pkgconfig
		
		# 5개의 ''libtcnative..' 파일들이 설치되었음을 확인할 수 있다.
		
	3. 톰캣 실행 환경변수에 arp/lib 경로 추가
		cd /usr/local/tomcat-9.0.113/bin
		vi setenv.sh
		..
		JAVA_OPTS="$JAVA_OPTS -Djava.library.path=/usr/local/apr-1.7.6/lib"
		export JAVA_OPTS
		:wq

============================= Tomcat Native Library 설치/end

6. 톰캣 server.xml 설정
	vi /usr/local/apache-tomcat-9.0.113/conf/server.xml
	
	<Listener className="org.apache.catalina.core.AprLifecycleListener" SSLEngine="on" />
	
	<!-- protocol="AJP/1.3" 을 protocol="org.apache.coyote.ajp.AjpAprProtocol" 변경 -->
	<Connector protocol="org.apache.coyote.ajp.AjpAprProtocol"
		address="127.0.0.1"	# worker.tomcat01.host=127.0.0.1 값과 일치해야함.
		port="8009"
		redirectPort="8443"
		maxParameterCount="1000" 
		secretRequired="true" 
		secret="1234"
		allowedRequestAttributesPattern=".*"  #요게 좀 확실히 알아봐야한다,명확하지 않다
       />
       
       <!--
	기본 http 커넥터 포트는 비활성화 처리, 다른 Connector 모두 비활성화 처리 
	<Connector port="8080" protocol="HTTP/1.1" connectionTimeout="20000" redirectPort="8443"/>
	또는
	<Connector port="8080" 
           protocol="org.apache.coyote.http11.Http11AprProtocol" <!--톰캣단독설정시, APR 프로토콜로 변경 -->
           connectionTimeout="20000"
           redirectPort="8443" />
	-->

7. 홈 디렉토리 설정

	아파치 documentroot 와 톰캣 appbase를 맞춰준다

	아파치 DocumentRoot 를 톰캣 기본 웹앱 디렉토리인 ROOT를 바라보게 한다.
	
	(httpd.conf)
	
		DocumentRoot "/home/tomcat/public/ROOT"
		<Directory "/home/tomcat/public/ROOT">

	(server.xml)
		<Host name="localhost"  appBase="/home/tomcat/public"
			    unpackWARs="true" autoDeploy="true">
			    
		- context 태그를 설정하지 않았다면 
		appBase(/home/tomcat/public)/ROOT 디렉토리가 기본 웹앱 홈이 된다.

===================

연동이 잘 되지 않는다면 로그를 잘 봐야한다.
	
	아파치 홈/logs/mod_jk.log # 아파치&톰캣 연동 에러 메시지가 여기에 있다.
	아파치 홈/logs/access_log # http 상태 정보를 확인할 수 있다.
	톰캣 홈/logs/catalina.out # jsp 로그를 볼 수 있다.
	톰캣 홈/logs/localhost_access_log # jsp access 로그를 볼 수 있다.
	
===================
SELinux 가 아파치 네트워크 연결을 차단하면 503에러가 발생한다.
SELinux 완전 끄던가. 
아니면 아래 명령으로 보안 설정을 한다
(아파치의 톰캣포트 AJP 8009로 통신 허용)

	setsebool -P httpd_can_network_connect 1
	
===================

계속 에러가 나서 헤맸는데, 제미나이에게 물어봐서 해결했다
	
	아파치홈/conf/workers.propeties 의
	
		worker.tomcat01.host=127.0.0.1	 #이거 중요
		
	와 톰캣 Connector 의 address 속성값을 일치시켜야 했다
	
		address="127.0.0.1"
