APR(Apache Portable Runtime)은 
Apache HTTP 서버 2.x 버전의 핵심 지원 라이브러리로, 
운영체제(OS) 간의 차이를 추상화하여 서로 다른 환경(Windows, Linux 등)에서도 
동일한 소스 코드로 작동할 수 있게 해주는 휴대용 라이브러리입니다. 

고급 I/O, 메모리 관리, 스레드 관리 등 시스템 수준의 기능을 일관된 API로 제공하여 
성능 향상과 플랫폼 독립성을 동시에 제공합니다

1. APR의 주요 목적 및 특징플랫폼 

	- 독립성 (Portability): 
		운영체제마다 다른 시스템 콜(System Call)을 APR이 일관된 API로 매핑하여, 
		개발자가 OS별로 코드를 다르게 작성할 필요 없이 이식성 높은 애플리케이션을 
		만들 수 있게 합니다.
		
	- 성능 최적화 (Performance): 
		고급 I/O 기능(예: sendfile, epoll, OpenSSL 등)을 사용하여 
		특히 웹 서버 환경에서 뛰어난 성능을 발휘합니다.
		
	- 핵심 기능 구현: 
		Apache 서버가 OS 수준에서 직접 수행해야 하는 
		파일 시스템 접근, 네트워크 프로그래밍, 프로세스/스레드 관리, 공유 메모리 
		등을 지원합니다.
	
2. APR 구성 요소
	
	- APR (Core): 
		파일 처리, 네트워크 소켓 등 운영체제 기본 기능을 다루는 핵심 라이브러리입니다.
	
	- APR-Util: 
		데이터베이스(DB) 접근, XML 파싱, 암호화 등 더 복잡한 도구 세트를 
		제공하는 라이브러리입니다.
		
3. 주요 장점

	에뮬레이션: 
		운영체제에서 특정 기능을 지원하지 않는 경우, 
		APR이 자체적으로 에뮬레이션하여 동일한 동작을 보장합니다.

	메모리 관리 (Pools): 
		APR 풀(Pools) 기술을 통해 메모리를 효율적으로 할당하고, 
		작업 종료 시 안전하게 해제하여 안정성을 높입니다.
	
4. 사용 사례

	Apache HTTP 서버: 
		2.x 버전 이후 핵심 엔진으로 사용되어 서버의 이식성을 책임집니다.
		
	Apache Tomcat: 
		Tomcat Native Library를 통해 
		APR을 사용하여 고성능 웹 서비스 환경을 구축합니다.
		
		Tomcat에서 APR 커넥터를 사용하려면 
		APR 라이브러리, 
		OpenSSL 라이브러리, 
		그리고 이를 연결하는 Tomcat Native Library가 필요합니다
