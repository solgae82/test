java 설치 후 톰캣 설치 진행한다 

.zip .tar.gz 등을 받으면 리눅스용 또는 윈도우용 모두 해당된다.
왜냐면 톰캣은 자바프로그램이므로..(jvm에서 돌아가는 프로그램)
widows.zip 등으로 되어 있는 것은 윈도우용 설치프로그램으로 설치되는 형식인 것이다.

설치는 압축 풀면 끝. 
나머진 설정

#톰캣 홈 디렉토리
	/usr/local/apache-tomcat-9.0.113

-------------------

1. tomcat 사용자 생성 
	
	:실행,정지,권한등 모든 액션은 tomcat 계정으로 할 수 있게 하기 위해
	
	adduser tomcat
	
2. 톰캣 홈 디렉토리, tomcat으로 권한 설정 

	sudo chown -R tomcat:tomcat /usr/localapache-tomcat-9.0.113 
	
3. java 홈 디렉토리, tomcat으로 권한 설정
	
	sudo chown -R tomcat:tomcat /usr/local/jdk-11.0.0.2
	
4. setenv.sh 생성 JAVA_HOME 설정
	
	(톰캣 8이상은 JAVA_HOME 환경변수만 설정해주면 된다.)
	
	su - tomcat # 톰캣계정으로 생성한다
	
	cd /usr/local/apache-tomcat-9.0.113/bin
	vi setenv.sh
	
		#!/usr/bin/bash
		export JAVA_HOME=/usr/local/jdk-11.0.0.2

	:wq
	
	sudo chmod 750 setenv.sh	#tomcat 권한인지도 확인.
	
5. 홈 디렉토리 설정
	(server.xml)
	
	cd /usr/local/apache-tomcat-9.0.113/conf
	
	# 설정 파일 백업
	cp server.xml server.xml.bak
	
	vi server.xml
	..
	<Host name="localhost"  appBase="/home/tomcat/public"
		    unpackWARs="true" autoDeploy="true">
	
	:wq
	
	---------------------
	- context 태그를 설정하지 않았다면 
	appBase(/home/tomcat/public)/ROOT 디렉토리가 기본 웹앱 홈이 된다.
	---------------------
	
6. firewall(방화벽에) 에 8080포트 허가 (root 계정으로 해야한다)
	
	# 현재 포트 현황 검색
	sudo firewall-cmd --permanent --list-ports # 열린 포트 목록 확인
	sudo firewall-cmd --list-services # 열린 서비스 목록 확인
	
	# 없으면 추가
	sudo firewall-cmd --permanent --zone=public --add-port=8080/tcp
	
	#재시작
	sudo firewall-cmd --reload 

7. 톰캣 시작/종료
	
	su - tomcat # 항상 tomcat 계정으로 실행한다.
	
	cd /usr/local/apache-tomcat-9.0.113/bin
	./startup.sh
	./shudown.sh

	# 기동 프로세스 확인
	ps -ef | grep tomcat
	
	# 8080 포트 확인
	netstat -nlt 
	
	
	http://해당IP:8080/ 테스트
	http://해당IP:8080/test.jsp 테스트
