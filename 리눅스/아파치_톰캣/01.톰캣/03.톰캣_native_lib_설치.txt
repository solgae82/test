
고성능 APR(Apache Portable Runtime) 기능을 쓰기 위해서
톰캣에서는 Tomcat Native Library 라는 JNI 라이브러리를 사용한다

	APR <- Tomcat Native Library(libtcnative)

그러므로 APR이 먼저 설치되어 있어야 한다.(그래야 쓰지)
그리고 (Tomcat Native Library)를 컴파일한 라이브러리를 
apr홈/lib에 설치하고 그 apr홈/lib 경로를 java 경로에 추가해서 
APR을 사용한다.

APR은 
아파치에도 설정해서 사용할 수 있고,(컴파일시 설정)
톰캣단독으로 설정해서 사용 할 수도 있다.

APR 설치 후 톰캣 연동 흐름은 아래와 같다.

톰캣 단독으로 설치할때는 
	
	1.apr(apr-util)설치  
	2.(Tomcat Native Library) 컴파일 설치
		apr홈/lib 에 'libtcnative*' 파일들 생성 확인.
	3.톰캣 실행 환경변수에 apr홈/lib 추가 설정
	4.톰캣 server.xml 설정 연동
	<Connector protocol="org.apache.coyote.http11.Http11AprProtocol" ../>	
	
아파치와 톰캣 연동할 때는 아파치 설치시 apr 연동 설치 되었을테니
	
	1.apr(apr-util) 설치 홈 경로 파악
	2.(Tomcat Native Library) 컴파일 설치.
		apr홈/lib 에 'libtcnative*' 파일들 생성 확인.
	3.톰캣 실행 환경변수에 apr홈/lib 추가 설정
	4.톰캣 server.xml 설정 연동
	<Connector protocol="org.apache.coyote.ajp.AjpAprProtocol" ../>
	

톰캣 Server.xml의 <Connector/> 태그 속성 protocol 값을 유심히 보면 
서로 다른 클래스명 설정이 보인다. 
(※이것이 중요한 핵심 포인트다)

APR 설치 후 
(Tomcat Native Library)를 다운로드 받아야 할텐데 어디서 다운로드 받을까?
톰캣홈/bin 에 가보면 'tomcat-native.tar.gz' 가 이미 포함되어 있다.
(톰캣 사이트에 가서 따로 특정 버전을 받아도 된다)

	톰캣홈/bin/tomcat-native.tar.gz

'tomcat-native.tar.gz' 파일이 (Tomcat Native Library) 소스 이다.
native 라이브러리이므로 압축 해제후 컴파일해서 설치해야한다.

아래는 톰캣 단독 설치시 
(Tomcat Native Library)를 설치하고 설정하는 과정을 정리했다.

아래 과정까지 성공해야, 아파치 연동시에도 편해진다.

(※ 아파치-톰캣 연동은 [02.아파치_톰캣연동] 디렉토리를 참조하자)

------------------- 1.apr(apr-util) 설치

1. 사전 도구 설치
	dnf -y install gcc gcc-c++ make expat-devel

2. 직접 필수 설치해야할 모듈 apr, apr-util, apr-iconv 다운로드 

	# dnf 보단 소스 컴파일해서 설치하도록 한다
	# 미리 검색해서 다운 받아 놓자
	cd /usr/loca/src
		apr-1.7.6.tar.gz
		apr-util-1.6.3.tar.gz
		apr-iconv-1.2.2.tar.gz

3. apr 설치
	
	cd /usr/local/src
	tar xvfz apr-1.7.6.tar.gz
	
	cd apr-1.7.6
	./configure --prefix=/usr/local/apr-1.7.6
	make && make install
	
	ls /usr/local # apr-1.7.6 디렉토리 생성되었는지 확인
	
4. apr-util 설치
	
	cd /usr/local/src
	tar xvfz apr-util-1.6.3.tar.gz
	
	cd apr-util-1.6.3/navtive
	
	./configure --prefix=/usr/local/apr-util-1.6.3 \
	--with-apr=/usr/local/apr-1.7.6
	
	make && make install
	
	ls /usr/local # apr-util-1.6.3 디렉토리 생성되었는지 확인
	

------------------- 2.(Tomcat Native Library) 설치 

1. (Tomcat Native Library) 설치 전 apr홈/lib 디렉토리를 확인해보자
	: 이곳에 접두사로 'libtcnative'* 가 붙는 라이브러리 파일이 추가 될 것이다
	
	ls /usr/local/apr-1.7.6/lib
		apr.exp     libapr-1.la  libapr-1.so.0      pkgconfig
		libapr-1.a  libapr-1.so  libapr-1.so.0.7.6

2. 컴파일 & 설치 
	
	cd /usr/local/tomcat-9.0.113/bin 
	tar xvfz tomcat-native.tar.gz
	ls 
		tomcat-native-1.3.1-src # .gz 압축해제 디렉토리 확인
		
	cd tomcat-native-1.3.1-src/native

	#설정 옵션을 자세히 보자 (검색해서 파악하면 좋다)
	./configure --prefix=/usr/local/apr-1.7.6 \
	--with-apr=/usr/local/apr-1.7.6/bin/apr-1-config \
	--with-java-home=/usr/local/jdk-11.0.0.2 \
	--with-ssl=yes

	# 컴파일 및 설치
	make && make install

3. 설치 잘 되었는지 확인 ('libtcnative..' 라이브러리 파일들 생성 확인)

	ls /usr/local/apr-1.7.6/lib
	
		apr.exp      libapr-1.so        libtcnative-1.a   libtcnative-1.so.0
		libapr-1.a   libapr-1.so.0      libtcnative-1.la  libtcnative-1.so.0.3.1
		libapr-1.la  libapr-1.so.0.7.6  libtcnative-1.so  pkgconfig
	
	# 5개의 ''libtcnative..' 파일들이 설치되었음을 확인할 수 있다.


------------------- 3.톰캣 실행시 java 환경변수에 apr홈/lib 추가 설정
 
두가지 설정법이 있다. (1번이 나은 방법)

1. JAVA_OPTS 환경변수에 라이브러리 경로 설정

	: JVM레벨 환경변수 JAVA_OPTS의 옵션 '-Djava.library.path=경로' 에 설정. 
	'/usr/local/apr-1.7.6/lib' 경로 추가
	
	cd /usr/local/tomcat-9.0.113/bin
	vi setenv.sh
	..
	JAVA_OPTS="$JAVA_OPTS -Djava.library.path=/usr/local/apr-1.7.6/lib"
	export JAVA_OPTS
	:wq

2. LD_LIBRARY_PATH (운영체제 라이브러리 경로) 환경변수에 설정

	: 운영체제 레벨 라이브러리 경로 환경변수인 'LD_LIBRARY_PATH'에 
	'/usr/local/apr-1.7.6/lib' 경로 추가
	
	cd /usr/local/tomcat-9.0.113/bin
	vi setenv.sh
	..
	# LD_LIBRARY_PATH 추가 설정
	LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/apr-1.7.6/lib
	export LD_LIBRARY_PATH		
	:wq	
	
	ldconfig
	
------------------- 4.톰캣 server.xml 설정 APR 연동

Tomcat AJP Connector 설정 변경 (server.xml)
: protocol="HTTP/1.1" 커네터를 찾아서
: protocol="org.apache.coyote.http11.Http11AprProtocol" 변경
	
	cd /usr/local/tomcat-9.0.113/conf
	vi server.xml
	..
	<!-- 톰캣 단독 서버 설정 -->
	<Connector port="8080" 
           protocol="org.apache.coyote.http11.Http11AprProtocol" #이전값: HTTP/1.1
           connectionTimeout="20000"
           redirectPort="8443" />
