Rocky 10 리눅스는 기본 모듈 중 pcre2를 써야한다!

최신 event MPM(Multi-Processing Module) 방식으로 설치

root 계정으로 설치 후 설치 디렉토리를 tomcat:tomcat 권한으로 모두 변경한다

1. 사전 도구 설치
	
	dnf groupinstall "Development Tools"
	dnf -y install gcc gcc-c++ make expat-devel openssl-devel
	
	#로키10은 pcre2 를 설치해야 한다 (pcre -> pcre2)
	dnf yum -y install pcre2-devel

2. 직접 필수 설치해야할 모듈 apr, apr-util, apr-iconv, pcre2, apache2.4 다운로드 

	# dnf 보단 소스 컴파일해서 설치하도록 한다
	# 미리 검색해서 다운 받아 놓자
	# 로키10은 pcre2 를 설치해야 한다 (pcre -> pcre2)
	# pcre2는 깃허브 zip으로 받으면 된다
	
	cd /usr/loca/src
		apr-1.7.6.tar.gz
		apr-util-1.6.3.tar.gz
		apr-iconv-1.2.2.tar.gz
		httpd-2.4.66.tar.gz
		pcre2-10.47.zip 

3. apr 설치
	
	cd /usr/local/src
	tar xvfz apr-1.7.6.tar.gz
	
	cd apr-1.7.6
	./configure --prefix=/usr/local/apr-1.7.6
	make && make install
	
	ls /usr/local # apr-1.7.6 디렉토리 생성되었는지 확인
	
4. apr-util 설치
	
	cd /usr/local/src
	tar xvfz apr-util-1.6.3.tar.gz
	
	cd apr-util-1.6.3
	./configure --prefix=/usr/local/apr-util-1.6.3 \
		--with-apr=/usr/local/apr-1.7.6 \
		--with-iconv
	make && make install
	
	ls /usr/local # apr-util-1.6.3 디렉토리 생성되었는지 확인
	
	
5. pcre2 설치
	
	cd /usr/local/src
	unzip pcre2-10.47.zip
	ls # pcre2-main 디렉토리가 방금 푼 pcre2-10.47.zip 이다
	
	cd pcre2-main
	./autogen.sh #이걸 해야 configure 파일이 생성된다.
	
	./configure --prefix=/usr/local/pcre2-10.47
	make && make install
	
	ls /usr/local # pcre2-10.47 디렉토리 생성되었는지 확인
	
6. openssl 위치 확인
	
	whereis openssl
	openssl: /usr/include/openssl
	
7. apache 설치
	
	cd /usr/local/src
	tar xvfz httpd-2.4.66.tar.gz
	
	cd httpd-2.4.66
		
	./configure --prefix=/usr/local/apache-2.4.66 \
	--enable-so \
	--enable-rewrite \
	--enable-proxy \
	--enable-expires \
	--enable-ssl \
	--with-ssl=/usr/include/openssl \
	--enable-mods-shared=all \
	--enable-mpms-shared=all \
	--with-mpm=event \
	--with-apr=/usr/local/apr-1.7.6 \
	--with-apr-util=/usr/local/apr-util-1.6.3 \
	--with-pcre2=/usr/local/pcre2-10.47/bin/pcre2-config

	# enable-mpms-shared=all, with-mpm=event 꼭 지정해라. 
	# (고성능 apr 사용과, 현대적 event MPM 방식)
	
	make && make install
	
	ls /usr/local # apache-2.4.66 디렉토리 생성되었는지 확인 
		
	
8. httpd.conf 설정
	
	cd cd /usr/local/apache-2.4.66/conf
	vi httpd.conf
	
		# ServerName 주석 풀고 'localhost' 설정
		ServerName localhost:80
		
		..
		
		LoadModule mpm_event_module modules/mod_mpm_event.so
		# LoadModule mpm_prefork_module modules/mod_mpm_prefork.so
		# LoadModule mpm_worker_module modules/mod_mpm_worker.so

		..

		# Various default settings
		Include conf/extra/httpd-default.conf #주석해제
		
		
		..
		# 웹홈 설정
		DocumentRoot "/home/tomcat/public/ROOT"
		<Directory "/home/tomcat/public/ROOT">
		..
		
	:wq
	
	cd cd /usr/local/apache-2.4.66/conf/extra
	vi httpd-default.conf
	
		#최적화 설정
		KeepAlive On
		MaxKeepAliveRequests 500
		KeepAliveTimeout 3
	
	:wq	
	---------------------
	KeepAliveTimeout 3: 
	요청 처리가 끝난 뒤 스레드 해제를 위해 대기하는 시간을 3초로 단축하여 
	리스너 스레드의 부하를 줄입니다
	---------------------
	
	웹홈 기본 테스트 설정시 :
	
		메인 호스트 방식:
		(conf/httpd.conf)
		
			DocumentRoot "/home/tomcat/public/ROOT"
			<Directory "/home/tomcat/public/ROOT">
			
		가상호스트 방식: 
		(conf/extra/httpd-vhosts.conf)
			
			<VirtualHost *:80>
			    ..
			    DocumentRoot "/home/tomcat/public/ROOT"
			    
	
9. httpd-mpm.conf 설정

	최신 Event MPM 을 사용하므로 아래 설정을 추가한다  
	(운영 서버 사양(RAM 8GB~16GB 기준)
	
	vi /usr/local/apache-2.4.66/conf/extra/httpd-mpm.conf
	<IfModule mpm_event_module>
	    # 서버가 시작할 때 생성할 초기 자식 프로세스 수
	    StartServers             5

	    # 최소 유지할 유휴(Idle) 스레드 수
	    MinSpareThreads          75

	    # 최대 유지할 유휴(Idle) 스레드 수
	    MaxSpareThreads          250

	    # 자식 프로세스 당 생성할 스레드 수
	    ThreadsPerChild          64

	    # 생성 가능한 최대 자식 프로세스 수 (MaxRequestWorkers / ThreadsPerChild)
	    ServerLimit              32

	    # 동시에 처리 가능한 최대 커넥션(요청) 수 (ServerLimit * ThreadsPerChild)
	    MaxRequestWorkers        2048

	    # 메모리 누수 방지를 위해 프로세스가 재시작되기 전 처리할 누적 요청 수 (0은 무제한)
	    MaxConnectionsPerChild   10000
	</IfModule>

	---------------------
	주요 값 산정 공식 가이드 (서버 사양 맞춤형)
	
	- MaxRequestWorkers는 
	[아파치에 할당할 가용 메모리 ÷ 아파치 프로세스 1개당 평균 메모리 사용량(약 50MB)] × ThreadsPerChild 
	공식으로 산정합니다.
	
	- 동시 접속자가 폭증하는 환경이라면 시스템 RAM 크기에 맞춰 
	ServerLimit와 MaxRequestWorkers를 비례하여 증설하십시오.
	---------------------

11. 방화벽 오픈

	firewall-cmd --permanent --list-ports # 열린 포트 목록 확인
	firewall-cmd --list-services # 열린 서비스 목록 확인
	
	#80번이 안 열려서, 직접 포트 지정 열고 싶다면..
	firewall-cmd --permanent --zone-public --add-port=80/tcp
	
	#방화벽 재시작
	firewall-cmd --reload
	
12. 기동/재시작/종료

	-k 옵션 : 원래 기동시 기본 옵션이다(대부분 생략해도 동작하게 되어있다고 한다)
	
	# 기동 테스트
	cd /usr/local/apache-2.4.66/bin	
	
	./apachectl -k start	# 시작
	./apachectl -k restart	# 재시작
	./apachectl -k stop	# 종료
	
	netstat -nlt #80번 포트 확인
	ps -ef | grep httpd #기동 프로세스 확인
	
	# MPM  모드 확인
	./httpd -V
	...
	Server MPM:     event
