apxs (APache eXtenSion)는 아파치 HTTP 서버의 확장 모듈을 컴파일하고 설치하는 도구로, 
DSO(Dynamic Shared Object)를 지원하는 아파치 환경에서 
모듈을 동적으로 로딩하여 사용할 수 있게 합니다. 

컴파일 뿐 아니라 컴파일한 .so 모듈 배치 및 
아파치 설정 파일(httpd.conf)에 모듈 추가 스크립트 추가 해준다

컴파일 및 설치 명령: 
	
	apxs -i -a -c mod_sample.c 
	
위 명령어를 사용하여 모듈을 
	-c : 컴파일 
	-i : 설치 , 아파치홈/modules/mod_sample.so
	-a : httpd.conf에 LoadModule 지시어를 자동으로 추가
	
컴파일 단계: 
	소스 코드(.c)를 아파치 모듈(.so)로 변환하는 과정으로, 
	apxs -c mod_sample.c 명령어를 통해 
	오브젝트 파일(.o)과 공유 객체(.so)를 생성합니다.
설치 단계: 
	apxs -i -a mod_sample.la 명령을 통해 
	컴파일된 모듈을 아파치 모듈 디렉터리로 복사하고 설정을 업데이트합니다.

필수 요건: 아파치 서버는 mod_so 모듈이 활성화되어 있어야 DSO 방식을 통한 모듈 로딩이 가능합니다. 

핵심 명령어 요약:
	컴파일: apxs -c [모듈명].c
	설치 및 자동 설정: apxs -i -a [모듈명].la
	한번에 실행: apxs -cia [모듈명].c 

이 도구는 주로 C 소스 파일(.c), 오브젝트 파일(.o), 라이브러리 아카이브(.a)를 다루며, 
자동으로 컴파일 및 링크 단계를 처리합니다. 

------------------- .la 파일이란

apxs(Apache eXtenSion) 도구를 사용하여 아파치 모듈을 컴파일할 때 생성되는 
.la 파일은 GNU Libtool 라이브러리 파일(Libtool Archive)입니다. 

주요 특징과 역할은 다음과 같습니다.

정의 및 역할: 
	Libtool이 생성한 텍스트 파일로, 실제 라이브러리(.so 파일)의 
	메타데이터와 의존성 정보를 담고 있습니다.

컴파일 정보 포함: 
	정적 라이브러리(.a)나 동적 라이브러리(.so)에 대한 의존성, 
	컴파일 관련 정보가 텍스트로 적혀 있어, 
	실제 모듈 설치 시 라이브러리 위치를 찾거나 연결하는 데 사용됩니다.
	
플랫폼 독립성: 
	
	여러 운영체제 간의 라이브러리 생성 차이를 줄여주기 위한 중간 매개체 역할을 합니다.
필요성: 
	보통 make install을 통해 모듈이 실제 아파치 모듈 디렉토리에 설치될 때 
	필요한 정보들이 포함되어 있습니다. 

요약하자면, .la 파일은 실제 아파치 모듈(.so)을 설치하고 연결하는 데 
필요한 설정 정보가 담긴 Libtool 기술 파일입니다.
