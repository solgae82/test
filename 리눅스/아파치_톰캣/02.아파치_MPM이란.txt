아파치 MPM은 Multi-Processing Module(다중 처리 모듈)의 약자로, 
아파치 웹 서버가 클라이언트의 요청을 받아들여 처리하는 방식을 결정하는 핵심 기능입니다. 

서버 성능과 자원(CPU, 메모리) 사용 방식을 정의하며, 주요 방식으로는 prefork, worker, event가 있습니다.

핵심 내용
역할: 요청 분배 및 처리 방식 정의 (자식 프로세스/스레드 관리).
목적: 웹 서버 성능 최적화 및 환경에 맞는 유연한 설정.

주요 MPM 방식:
	Prefork: 프로세스 기반. 안정적이지만 메모리 사용량이 많음.
	Worker: 프로세스+스레드 혼합. 효율적이고 많은 요청 처리 가능.
	Event: Worker와 유사하나 Keep-Alive 연결 처리 개선으로 고성능 제공.
	
웹 애플리케이션의 성격과 서버 사양에 따라 최적의 MPM 모듈을 선택하여 사용합니다.

-------------------

아파치(Apache)의 Prefork, Worker, Event 방식은 클라이언트의 
요청을 처리하는 멀티 프로세싱 모듈(MPM)의 발전 단계입니다.
	
	Prefork -> Worker -> Event
	
가장 핵심적인 차이는 Keep-Alive(지속적 연결) 상황에서 
스레드 자원을 얼마나 효율적으로 해제하고 분배하는가에 있습니다.


어떤 방식을 선택해야 할까?

Prefork 모드 선택:
	사용 중인 외부 라이브러리나 PHP 모듈(mod_php)이 
	Thread-Safe(스레드 안전)하지 않을 때 필수적입니다.
	동시 접속자가 많지 않고 시스템 안정성이 절대적으로 중요할 때 사용합니다.

Worker 모드 선택:
	Apache 2.2 이하의 구버전 환경이면서 대용량 처리가 필요할 때 적합합니다. 
	(현재는 대부분 Event로 대체됨)

Event 모드 선택 (현대 표준):
	Apache 2.4 버전 이상의 기본값입니다.
	대규모 동시 접속자가 몰리는 고트래픽 서비스 환경에 필수적입니다.
	동적 처리가 필요하다면 Apache와 PHP를 직접 묶지 않고, 
	[Apache (Event MPM) + PHP-FPM] 구조로 연동하여 성능을 극대화합니다.
