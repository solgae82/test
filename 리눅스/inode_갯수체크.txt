df -i 명령어는 파티션별 디스크 공간 대신 
아이노드(Inode) 사용량을 확인하는 명령어입니다. 

리눅스 파일 시스템은 파일마다 하나의 아이노드를 할당하므로, 
디스크 용량이 남아있어도 아이노드가 고갈되면 더 이상 파일을 생성할 수 없습니다.

df -i 명령 출력시 각 항목의 상세 의미는 다음과 같습니다.

📊 df -i 출력 항목 상세 설명

	Inodes (아이노드 총량)
		해당 파일 시스템이 생성할 수 있는 최대 파일 및 디렉터리 개수입니다.
		파일 시스템 포맷 시점에 디스크 크기에 비례하여 자동 할당됩니다.

	IUsed (사용 중인 아이노드)
		현재 디스크에 생성되어 있는 파일 및 디렉터리의 총개수입니다.
		파일의 크기와 상관없이 파일 1개당 아이노드 1개가 소비됩니다.
		
	IFree (남은 아이노드)
		앞으로 추가로 생성할 수 있는 파일 및 디렉터리의 개수입니다.
		
		이 수치가 0에 가까워지면 디스크 용량이 남아있어도 파일 생성 시 
		"No space left on device" 오류가 발생합니다.
		
	IUse% (아이노드 사용 비율)
		전체 아이노드 중 현재 사용 중인 아이노드의 백분율(%)입니다.
		이 수치가 90% 이상을 넘어가면 시스템 경고로 인식하고 
		불필요한 파일을 정리해야 합니다.
	
💡 주요 점검 팁

용량은 남았는데 파일 생성이 안 될 때: 
	df -h에서는 용량이 여유롭지만 df -i에서 IUse%가 100%라면, 
	아주 작은 크기의 파일(로그, 세션 파일 등)이 수백만 개 쌓여있는 상태입니다.
	
해결 방법: 
	/tmp, /var/spool, 세션 저장 경로 등에서 
	불필요한 수많은 소형 파일들을 찾아 삭제해야 합니다.
