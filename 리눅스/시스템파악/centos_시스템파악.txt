3. 네트워크 설정
	
	네트워크 설정: IP, 서브넷 마스크, 게이트웨이, DNS 설정 (ip a, nmcli)
	
	호스트네임 및 IP 주소: hostname, ip addr (또는 ifconfig) 명령어로 네트워크 식별 정보 확인. 
		hotsname 
			localhost.localdomain
		
		ip addr		#ip, (/24) netmask, broadcast
			inet 192.168.0.6/24 brd 192.168.0.255
		
		ifconfig
			inet 192.168.0.6  netmask 255.255.255.0  broadcast 192.168.0.255
			
	게이트웨이 확인(라우팅 테이블): ip route 명령어로 기본 게이트웨이를 확인합니다. 

		route -n # Destination '0.0.0.0' 인게 현재 네트워크 인터페이스
		
			Kernel IP routing table
			Destination     Gateway         Genmask         Flags Metric Ref    Use Iface

			169.254.0.0     0.0.0.0         255.255.0.0     U     1002   0        0 enp3s0

			192.168.0.0     0.0.0.0         255.255.255.0   U     0      0        0 enp3s0
			192.168.0.0     0.0.0.0         255.255.255.0   U     0      0        0 enp4s0
			192.168.122.0   0.0.0.0         255.255.255.0   U     0      0        0 virbr0
		
		ip route	# enp3s0 인터페이스

			169.254.0.0/16 dev enp3s0 scope link metric 1002
			169.254.0.0/16 dev enp4s0 scope link metric 1003

	랜선 연결된 네트워크 인터페이스 찾기
		
		#'enp3s0' 네트워크 인터페이스 기준으로 검색시 
		cat /sys/class/net/enp3s0/carrier # 1:랜선연결, 0:연결안됨
		또는 
		cat /sys/class/net/enp3s0/operstate # UP:랜선연결, down:연결안됨
	
		# ip 명령어로 랜선연결 네트워크 인터페이스 찾기
		ip -br link # 네트워크 인터페이스 중 'UP' 표시가 된 것이 현재 랜선연결된 네트워크 인터페이스

5. 보안 및 사용자 관리

	Root 권한 사용자: cat /etc/passwd로 시스템 사용자 목록을 확인합니다.
	
	사용자 및 권한: 루트 계정 관리 및 sudo 권한이 있는 사용자 명단 (/etc/passwd, /etc/sudoers)
	
	
		# sudo 권한 계정 파악
		/etc/sudoers
			..
			root    ALL=(ALL)       ALL
			dong    ALL=(ALL)       ALL
			..
	
	SSH 접속 설정: /etc/ssh/sshd_config 파일에서 Root 로그인 허용 여부, SSH 포트 번호를 확인합니다.
	
	방화벽 설정: iptables -L 또는 firewall-cmd --list-all (또는 ufw status) 명령어로 방화벽 규칙을 확인합니다.

6. 로그 및 백업 (가장 중요)

	백업 상태: 가장 중요한 정보로, 백업 자동화 스크립트(cron) 및 백업 데이터의 위치/무결성 확인.
	
	로그 위치: /var/log/ 디렉토리 내의 주요 로그 파일(messages, secure, syslog) 위치와

		백업 데이터가 정상적으로 저장되고 있는지 확인합니다.
