
setcap은 리눅스 파일에 특정 권한(Capability)을 부여하여, 
일반 사용자도 root 권한 없이 1024 이하 포트 바인딩 등 제한된 작업을 수행할 수 있게 하는 명령어입니다. 
보안을 위해 불필요한 root 권한 상승 대신 파일 단위의 세분화된 권한을 부여할 때 사용합니다


목적: 바이너리 파일에 특정 기능(Capability) 부여.
핵심 용도: non-root 사용자가 1023 이하의 Well-known 포트를 사용해야 할 때.

주요 옵션 및 기능
	+ep: 권한을 추가 (e: effective, p: permitted).
	-r: 파일에 부여된 모든 capability를 제거.
	-v: 파일의 capability를 검증 (verify)
	
권한 설정: setcap 'cap_net_bind_service=+ep' /usr/local/apache2.4.66/bin/httpd
권한 삭제: setcap -r /usr/local/apache2.4.66/bin/httpd
권한 확인: getcap /usr/local/apache2.4.66/bin/httpd

------------------- (예제)
#1024 이하 포트는 root 만 실행가능한데, 일반사용자도 실행할 수 있게 변경

sudo setcap 'cap_net_bind_service=+ep' /usr/local/apache2.4.66/bin/httpd

------------------- setcap 명령어 옵션 분석
	
setcap: 파일에 능력(Capability)을 설정하는 명령입니다.

cap_net_bind_service: 
	1024 미만의 특권 포트(Privileged ports)에 
	소켓을 바인딩할 수 있게 해주는 Linux 능력.

+ep:
	+: 뒤에 오는 능력(Capability)을 추가한다는 의미입니다.
	e (Effective): 실행 시 바로 효력이 발생하는 상태입니다.
	p (Permitted): 해당 능력을 사용할 수 있도록 허용된 상태입니다.
	
/usr/local/apache2.4.66/bin/httpd : 설정할 바이너리 프로그램

즉, 해당 프로그램이 실행될 때 1024 미만 포트 사용 권한 부여.

주의사항 
	이 명령은 파일(바이너리)에 직접 권한을 부여하는 것이므로, 스크립트 파일에는 작동하지 않을 수 있습니다.

능력이 설정된 파일은 getcap [파일경로] 명령어로 확인할 수 있습니다

	sudo getcap /usr/local/apache2.4.66/bin/httpd
	[sudo] tomcat 암호:
	/usr/local/apache2.4.66/bin/httpd cap_net_bind_service=ep
