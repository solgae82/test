데이터 모델링
	
	현실세계 -> 데이터베이스 추상화
	고객의 업무 프로세서 이해 후 모델링 표기법으로 모델링

	데이터 모델링의 특징
		추상화(Abstraction)
		단순화(Simplification)
		명확화(Clarity)

	데이터 모델링의 단계
		개념적 모델링
			- 추상화가 가장 높다
			- ERD 작성
		논리적 모델링
			- 식별자 도출, 릴레이션 정의, 정규화
		물리적 모델링
			- 데이터 베이스 구축

	데이터 모델링의 관점(View)
		데이터
			- 구조분석,정적분석 
		프로세스
			- 시나리오분석, 도메인 분석, 동적 분석
			
		데이터와 프로세스
			- CRUD 분석

	ERD 작성절차
		1.엔터티 도출
		2.엔터티 배치
		3.엔터티 관계 설저
		4.관계명 기술
		5.관계 참여도 표현(관계수)
		6.관계 필수 여부 표현

	데이터 모델링 고려사항
		데이터 모델의 독립성
		고객 요구사항의 표현
		데이터 품질 확보

	3층 스키마(3-level schema)
		스키마 구조
			외부스키마(사용자관점)
				- 응용프로그램 접근 DB
			개념스키마(설계자관점)
				- 전체집단의 데이터베이스 구조(통합 데이터베이스)
			내부스키마(개발자관점)
				- 물리적 저장 구조

		독립성
			논리적 독립성 : 개념 스키마 변경 -> 외부 스키마 영향 없음
			물리적 독립성 : 내부 스키마 변경 -> 개념스키마 영향 없음

엔터티(Entity)
	- 저장되고 관리되어야 하는 데이터 집합
	- 개념, 사건, 장소등의 명사이다(Thomas Bruce)

	엔터티 의미
		Peter Chen : 엔터티는 변별할 수 있는 사물이다
		James Martin :  정보를 저장할 수 있는 어떤 것
		C.J Date : 데이터베이스 내부에서 변별 가능한 객체이다.
		Thomas Bruce : 정보가 저장될 수 있는 장소,사람,사건,개념,물건 등이다.

	엔터티 도출
		고객의 비즈니스 프로세스의 관리되어야 하는 정보에서 추출해야 한다

	엔터티 특징
		식별자 : 유일한 식별자가 있어야 한다
		인스턴스 집합: 2개 이상의 인스턴스가 있어야 한다
		속성 : 반드시 속성을 가지고 있어야 한다
		관계 : 다른 엔터티와 최소한 한개 이상 관계가 있어야 한다.
		업무 : 업무에서 관리되어야 하는 집합이다

	엔터티 종류
		유형,무형에 따른 종류
			유형엔터티
				-물리적 형태가 있는 엔터티
				-고객,강사,사원 등
			개념엔터티
				-물리적 형태가 없는 엔터티 (개념적 엔터티)
				-거래소 종목,생명보험 상품 등
			사건엔터티
				- 비즈니스 프로세서 실행에 의해 생성되는 엔터티
				- 주문,체결, 청구 등

		발생 시점에 따른 종류
			기본 엔터티(key entity, basic entity)
				- 독립적 엔터티
				- 고객,상품,부서 등
			중심 엔터티(Main Entity)
				- 기본 엔터티로부터 발생되고 행위 엔터티를 생성하는 것
				- 계좌,주문,취소,체결 등
			행위 엔터티(Active Entity)
				- 2개 이상의 엔터티로부터 발생
				- 주문이력, 체결 이력 등
속성(Attribute)
	- 엔터티 구성 항목
	- 인스턴스 구성 요소 
	- 더 이상 분해 안된다

	속성의 특징
		- 업무에서 관리되는 정보
		- 일반적으로 하나의 값만 가진다
		- 주 식별자에게 함수적으로 종속된다.

	속성의 종류
		분해 여부에 따른 종류
			단일 속성
				- 회원ID, 이름 등
			복합 속성
				- 주소는 시,군,동 등으로 분해 된다
			다중값 속성
				- 상품리스트
				- 다중값 속성은 엔터티로 분해된다.

		특성에 따른 종류
			기본 속성 
				- 본래의 속성
				- 회원ID,이름 등
			설계 속성 
				- 데이터 모델링 과정에성 발생되는 속성
				- 유일값을 가진다
				- 상품코드, 지점코드 등
			파생 속성
				- 다른 속성에서 만들어지는 속성
				- 합계, 평균 등

		※ 도메인
			- 속성이 가지는 값의 범위
			- 남,여 등
관계(relationship)
	관계의 종류
		존재 관계
			- 엔터티 간의 상태
			- 고객 -> 관리점 할당
		행위 관계
			- 엔터티 간의 행위
			- 계좌 -> 주문

	관계 차수 
		최대 기수성
			일대일(1:1)
				- 완전 1대1
				- 선택적 1대1
			일대다(1:N)
			다대다(M:N)
				- 카테시안 곱 발생
				- 1:1 또는 1:N 관계로 해소해야 함.
		최소 기수성
			- 필수 ('|'), 선택 (○)
	식별관계와 비식별 관계
		강한 개체 : 독립적 개체
		약한 개체 : 강한 개체의 존재에 의존되는 개체
		
		식별 관계 
			- 강한 객체의 기본키 공유시 약한객체 기본키의 하나로 관계 가짐.
			- 실선 표현
		비식별 관계
			- 강한 객체의 기본키를 공유시 약한 객체 일반 컬럼으로 관계 가짐.
			- 점선 표현

엔터티 식별자(Entity Identifier)
	주식별자(기본키, Primary Key)
		- 최소성
		- 대표성
		- 유일성
		- 불변성

		키 종류
			- 후보키(Candidiate key) : 유일성과 최소성을 만족하는 키
			- 기본키(Primary key): 후보키 중 엔터티를 대표할 수 있는 키
			- 슈퍼키(Super key): 유일성 만족, 최소성 만족 않는 키
			- 대체키(Alternate key): 후보키 - 기본키 = 대체키
			- 외래키(Foreign key): 다른 Entity의 기본키, 참조 무결성을 위한 키
	식별자 종류

		식별자의 대표성
			주 식별자
			보조 식별자
		생성 여부
			내부 식별자
			외부 식별자
		속성의 수
			단일 식별자
			복합 식별자
		대체 여부 
			본질 식별자
			인조 식별자
