정규화
	데이터 일관성
	최소한의 데이터 중복
	최대한의 데이터 유연성
	데이터 분해(테이블 분해)

	정규화 절차
		제1정규화
			- 원자성 확보,기본키 설정

		제2정규화
			- 기본키가 하나면 제2정규화는 생략
			- 기본키가 2개 이상 속성일 경우
			- 부분함수 종속성 제거

		제3정규화
			- 기보키를 제외한 컬럼 간에 종속성 제거
			- 이행함수 종속성 제거
		BCNF
			- 후보키가 기본키를 종속시키면 분해
		제4정규화 
			- 다중값 종속성 제거
		제5정규화
			- 조인 종속성 분해

반정규화 (De-Normalization)
	정규화는 조인을 발생 시키는데,
	잦은 조인으로 인한 성능 저하시 거꾸로 반정규화를 한다 
	조인을 줄이는 성능 향상 기법
	select 속도는 향상하나, 데이터 모델의 유연성은 낮아진다.

	반정규화 절차
		- 대상 조사 및 검토
		- 다른 방법 검토
		- 반정규화 수행
	반정규화 기법
		- 계산된 컬럼 추가
		- 테이블 수직 분할 : 컬럼 분할
		- 테이블 수평 분할 : row의 특정 값을 기준으로 분할
			파티션 분할
				- Range Partition : 특정 범위
				- List Partition : 특정값
				- Hash Partition : 해시 함수 사용
				- Composite Partition : 범위와 해쉬를 복합적으로 사용
	테이블 병합
		1대1, 1대N, 슈퍼타입 + 서브타입 : 통합

		슈퍼타입과 서브타입
			부모 자식 관계
			배타적 관계, 포괄적 관계가 있다
			
			변환방법
				- OneToOne Type : 
					- 슈퍼타입 서브타입을 개별 테이블로 도출
					- 테이블 수가 많아서 조인이 많이 발생, 관리 어려움
				- Plus	Type
					- 슈퍼타입 서브타입으로 도출
					- 조인이 발생, 관리 어려움(onetone type보다 나음)
				- Single type 
					- 슈퍼타입과 서브타입을 하나의 테이블로 도출
					- 조인 성능 좋고, 괸리 편하지만 입출력 성능 저하
분사데이터베이스
	
	분산데이터베이스 투명성 종류
		분할 투명성
		위치 투명성
		지역 사상 투명성
		중복 투명성
		장애 투명성
		병행 투명성

	분산데이터베이스 설계 방식
		상향식 설계 방식
			지역 스키마 -> 전역 스키마
		하향식 설계 방식
			전역스키마 -> 지역 사상 스키마

	분산데이터 베이스 장단점
		장점
			- 신뢰성,가용성 높음
			- 병렬처리로 빠른 응답
			- 용량 확장 용이
		단점
			- 관리와 통제 어려움
			- 보안 관리 어려움
			- 데이터 무결성 관리 어려움
			- 설계 복잡
