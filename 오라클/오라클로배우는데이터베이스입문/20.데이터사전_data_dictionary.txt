오라클 테이블은 사용자 테이블(user table),데이터 사전(data dictionary)으로 크게 나뉜다

사용자 테이블은 Nomal Table , 데이터 사전은 Base Table 이라고도 부른다.

오라클 데이터사전(dictionary)에는 데이터베이스 메모리,성능,사용자,권한,객체 등
데이터베이스 운영에 대한 중요정보가 보관되어 있다.

오라클 데이터베이스는 사용자가 데이터사전에 직접 접근하는 것을 막고 있다.
대신, 데이터사전 뷰(data dictionary view)를 제공하여 select문의 접근만 허용한다.
아래 SQL문으로 데이터사전 테이블 뷰들을 검색할 수 있다

	select * from DICT; -- 축약
	또는
	select * from DICTIONARY; 

SQL문을 실행하면 TABLE_NAME 컬럼에 나오는 정보가 데이터 사전 뷰 테이블 정보이다.
이 사전 뷰 테이블 이름을 보면 패턴 접두사가 붙는데, 의미는 아래와 같다
	
	- USER_xxxx : 현재 데이터베이스에 접속한 사용자가 소유한 객체 정보 사전 뷰 테이블
	
	- ALL_xxxx : 현재 데이터베이스에 접속한 사용자가 소유한 객체와 다른 사용자가 소유한 객체
	중 사용 허가를 받은 객체. 즉, 사용 가능한 모든 객체정보 사전 뷰 테이블
	
	- DBA_xxxx : 데이터베이스 관리를 위한 정보 사전 뷰 테이블(sys, system 만 접근 가능)
	
	- V$_xxxx : 데이터베이스 성능 관련 정보(X$_xxxx 테이블의 뷰)

너무 많이 있지만, 이 중 중요 사전 뷰 테이블을 살펴 보면서 안목을 키워나가면 된다.

	--사용자 소유 테이블들(USER_TABLES)
	select * from USER_TABLES;
	select table_name,tablespace_name,num_rows from USER_TABLES;

	-- 사용자 소유 테이블들 + 사용 허가된 테이블들 (all_tables)
	select owner, table_name, num_rows from all_tables 
	where owner='DONG' and  table_name = 'EMP';

	-- 사용자 소유 테이블 변경 이력, USER_TAB_MODIFICATIONS:(insert, delete)
	SELECT TABLE_NAME, INSERTS, DELETES
	FROM USER_TAB_MODIFICATIONS;
	
	: 아래는 DBA_xxx 가 붙는 (sys, system 사용자만 접속 가능한) 사전 뷰 테이블 
	
	-- 데이터 베이스 모든 테이블 정보 보기
	select * from DBA_TABLES;

	-- 데이터베이스 모든 사용자 정보 보기
	select * from DBA_USERS;

---- etc

수천억건의 행이 있는 대용량 테이블을 count(*) 로 조회하면 엄청난 시간이 걸릴 것이다.
이럴때 테이블 정보를 보여주는 사전 뷰 테이블을 이용하면 빠르게 행수를 알아내는 방법도 가능해진다.

예를 들면,
USER_TABLES 테이블의 NUM_ROWS 컬럼과, 
USER_TAB_MODIFICATIONS 테이블의 INSERTS, DELETES 컬럼을 연산하면 대용량 테이블의 행수를 
빠르게 알아낼 수 있다

(테이블행수 빠르게 파악하기)
	SELECT 
		T.TABLE_NAME, 
		T.LAST_ANALYZED, 
		T.NUM_ROWS, 
		NVL(M.NUM_ROWS,0) ADD_ROWS,		
		T.NUM_ROWS+NVL(M.NUM_ROWS,0) TOT_ROWS
	FROM 
	USER_TABLES T 
	LEFT OUTER JOIN 
	(

		    SELECT
			TABLE_NAME,
			( INSERTS -DELETES ) NUM_ROWS
		    FROM
			USER_TAB_MODIFICATIONS
		    WHERE
			PARTITION_NAME IS NULL

	) M

	ON T.TABLE_NAME = M.TABLE_NAME
	WHERE T.TABLE_NAME ='EMP'
	;
