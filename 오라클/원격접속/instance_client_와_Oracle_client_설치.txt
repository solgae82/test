----------------------------------- instance 클라이언트로 설정하기
1.instance client 다운로드
	-instance client 다운로드 주소(32bit용)
	https://www.oracle.com/technetwork/topics/winsoft-085727.html

	- 두개 다운로드 받는다 (32bit 용은 파일명에 'nt'가 붙는다)
	1) instantclient-basic-nt-11.2.0.4.0.zip (basic)
	2) instantclient-sqlplus-nt-11.2.0.4.0.zip(sqlplus)

	-----------------------------------------//	
	특정 버전 (예를 들어 12.2)의 경우 basic 파일 옆 Descripton란에 아래 같은 글이 있고 링크가 있다
	(링크가 깨져 있어서 구글 검색으로 다운 받아야 한다)
	
	"The 12.2 Basic package requires the Microsoft Visual Studio 2013 Redistributable."
	
	이게 뭐냐면 
		Visual C++로 개발된 애플리케이션이 Visual C++가 설치되지 않은 컴퓨터에서도 
		실행될 수 있도록 하는 패키지 (C런타임,표준C++라이브러리등등) 
		(java JRE 같은 느낌)
	저 재배포 패키지가 설치되어 있어야 해당 프로그램이 동작할 수 있다. 그러므로 설치해줘야 함.
	
	32bit instance client 설치 문서에 보면 이런식으로 나와 있다
	
		Microsoft에서 올바른 Visual Studio 재배포 가능 패키지를 다운로드하여 설치하세요. 
		- Instant Client 12.2에는 Visual Studio 2013 재배포 가능 패키지가 필요합니다. 
		- Instant Client 12.1에는 Visual Studio 2010 재배포 가능 패키지가 필요합니다.		
		
	구글링으로 검색해서 Microsoft 다운로드 센터에 가서 프로그램 다운로드 후 실행하면 됨. 
		1.vcredist_x86.exe
		2.credist_x64.exe 
	다운로드 후 실행! 설치
	
	(운 좋게 저 재배포 패키지를 설치 하지 않아도 동작하는 경우는 visual studio가 설치 되었거나,
	기타 MS OFFICE 같은 프로그램이 설치되어서 해당 재배포 패키지가 이미 설치되었을 때의 경우이다)

2. 설정 디렉토리에 설치
	디렉토리 만들기 : C:\oracle\11.2.0\client_1 
	C:\oracle\11.2.0\client_1\ 디렉토리 안에  다운로드 받은 zip파일 모두 풀어서 넣는다
	
3. 환경변수 ORACLE_HOME 만들고 PATH 추가 
	환경변수 추가 => ORACLE_HOME : C:\oracle\11.2.0\client_1
	PATH 환경변수에 추가  => ;%ORACLE_HOME%
	
4. network\admin 디렉토리 만들기
	: 네트워크 설정 파일들 디렉토리로 tnsnamse.ora 파일과 sqlnet.ora 파일을 넣을 공간 마련
	
	C:\oracle\11.2.0\client_1\network\admin

5. 환경변수 추가 (TNS_ADMIN , NLS_LANG) , (선택사항)
	TNS_ADMIN : %ORACLE_HOME%\network\admin
	NLS_LANG : KOREAN_KOREA.UTF8 (오라클 서버와 동일하게 설정)
	
6. tnsnames.ora 파일 작성
	ora10 =
	  (DESCRIPTION =
	    (ADDRESS_LIST =
	      (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.0.3)(PORT = 1521))
	    )
	    (CONNECT_DATA =
	      (SERVER = DEDICATED)
	      (SERVICE_NAME = ora10)
	    )
	  )

7.sqlnet.ora 파일 작성
	SQLNET.AUTHENTICATION_SERVICES= (NTS)
	NAMES.DIRECTORY_PATH= (TNSNAMES, ONAMES, HOSTNAME)

--------------------------------  Oracle Client 설치
setup.ext 를 실행하면
위 3번 과정까지 자동으로 진행되고
4번부터 수동으로 만들면 된다.

-------------------------------- 환경변수
실험해보니 
	1) instance client경로\network\admin 만든 후 *.ora 파일을 설정
	2) 환경변수 추가 (2가지 방법)
		- 시스템 Path경로에  instance client경로를 추가
			Path=C:\oracle\11.2.0\client_1
		
		- ORACLE_HOME 환경변수 만들고 instance client 경로 설정
			ORACLE_HOME=C:\oracle\11.2.0\client_1
			
환경변수 추가 하는 방법 둘 중 하나만 해도 토드나, sql developer에서 TNS 방식 잘 읽어온다
쉽게 말하면 instance client 경로 설정 방법이 2개 인것이다.
물론 두 가지 다 설정 해도 동작한다.

위와 같이 설정할 instance client경로 안에 network\admin 을 만들고 그 안에 *.ora 설정 파일을 만들면 
TNS_ADMIN 환경변수 만들지 않아도 접속 프로그램들이 그 경로에 설정파일들을 자동으로 읽어오게 약속되어 있다.

특별히 설정파일 경로를 다른 경로로 설정해서 관리하고자 할 경우 
명시적으로 환경변수 TNS_ADMIN 만들어 사용하도록 약속되어 있는 듯 하다
아마도,
	1)intace client경로\network\admin\*.ora  설정 파일들이 있는지 찾아보고 없으면
	2)TNS_ADMIN 환경변수 경로를 찾아본다

그러므로 최종 요약하면, 
	1)시스템 Path 경로에 intace client경로 추가
	2)intace client경로\network\admin\*.ora 설정
	하면 끝.
	
	(요즘은 이렇게 설정하는 것 같고, 
	옛날 버전 프로그램들은 저 환경 변수들 'ORACLE_HOME','TNS_ADMIN'이 필요했던거 같다)
