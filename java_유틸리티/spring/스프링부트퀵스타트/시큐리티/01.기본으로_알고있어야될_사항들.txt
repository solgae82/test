스프링 시큐리티 기본 설정은 모든 요청에 인증을 요구하며, 
기본 사용자 아이디는 user, 비밀번호는 애플리케이션 실행 시 콘솔에 출력되는 무작위 UUID입니다. 
폼 로그인(formLogin) 및 HTTP 기본 인증(httpBasic)이 활성화되며, CSRF 보호 기능이 기본으로 작동합니다. 

인증 필수: 모든 URL 요청은 인증을 받아야 합니다.
기본 사용자: 아이디는 user, 비밀번호는 서버 콘솔에서 확인 가능한 랜덤한 값.
로그인 방식: /login 페이지(기본 폼 로그인) 및 HTTP Basic 인증 제공.
보안 기능: CSRF 보호가 활성화되어 POST 요청 등에 토큰 필요.
로그아웃: 기본 로그아웃 URL은 /logout이며 세션 및 쿠키 삭제. 
이러한 기본 설정은 SecurityFilterChain을 커스터마이징하여 변경할 수 있습니다. 

권한이름 : 자동으로 접두사 'ROLE_' 이 붙어서 'MEMBER' 권한이면, 이름은 'ROLE_MEMBER' 가 된다.(시큐리티 7.0.2)
	
	- 시큐리티 7.0.2 (ROLE 이라는 접두사를 억지로 붙이면 예외 발생함)
	- 또 어떤 블로그 문서에서는 'ROLE' 붙여서 사용해야한다고 하기도 하는등. 좃같은 상황이다
	
	
연결 처리의 대부분이 약속된 프로토콜로 숨겨져 있으므로 잘 때려 맞춰야 한다.

시큐리티의 약속된 프로토콜에 맞춰 로그인 관련 정보를 사용자가 잘 제공해주기만 하면, 
나머지 로직은 자동으로 진행된다.
그 자동진행 부분을 잘 유추해서 내 것으로 만들어야, 다음 단계에서 헤매지 않게 된다.

참 좃같은 방식이라 아니할 수가 없다.
