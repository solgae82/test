<파이썬 내부 인코딩 방식>

자바는 묻지도 따지지도 않고 내부에서 무조건 2바이트(최신 버전은 영문일 때만 1바이트) 고정 형태를 쓰지만, 
파이썬은 문자열의 
'최대 코드포인트'를 검사해서 내부 C 구조체의 '글자당 방 크기(1, 2, 4바이트)' 자체를 
통째로 바꾸는 가변 방식을 씁니다 ( 글자당 크기가 변하는 가변 메커니즘 )

파이썬 3부터 모든 문자열은 사람이 읽을 수 있는 문자를 추상적으로 표현하는 유니코드(Unicode) 객체입니다.

1. 파이썬 내부의 3가지 가변 유니코드 

	파이썬은 메모리를 극단적으로 아끼기 위해 문자열에 포함된 글자 중 '가장 번호(코드포인트)가 큰 글자'를 
	기준으로 문자열 전체의 방 크기를 결정합니다.
	
	Latin-1 구조체 (PyASCIIObject / PyCompactUnicodeObject)
	
	문자열에 아스키/Latin-1(코드포인트 0~255)만 있을 때 선택됩니다.
	글자당 딱 1바이트만 씁니다.
	
	UCS-2 구조체
	문자열에 한글, 한자 등 기본 유니코드 영역(코드포인트 256~65535)이 섞여 있을 때 선택됩니다.
	글자당 2바이트를 씁니다.
	
	UCS-4 구조체
	문자열에 이모지(\ud83d\ude00)나 희귀 고전 문자(코드포인트 65536 이상)가 단 하나라도 섞이면 선택됩니다.
	글자당 4바이트를 씁니다.
	
	
2. 파이썬 내부 문자열 처리

	파이썬 버전별로 매우 다르다.
	
	(3.12버전 테스트)
	import sys

	str_data = 'python'

	# 1. 글자당 몇 바이트짜리 구조체를 썼는지 확인 (기본 문자 크기)
	# sys.getsizeof() 공식 문서에 따른 문자 속성 검사 방식으로 추정 가능
	print(f"글자 수: {len(str_data)}글자")
	print(f"실제 메모리 총 크기: {sys.getsizeof(str_data)} 바이트")
	
	'''
	글자 수: 6글자
	실제 메모리 총 크기: 47 바이트
	'''
		
	[파이썬 내부에서 일어나는 일]
	
	1. 파이썬은 'python'이라는 문자열을 조사합니다. 전부 알파벳(아스키 영역)뿐입니다.
	2. "오케이, 이건 1바이트짜리 Latin-1 구조체로 만든다!" 하고 결정합니다.
	3. 내부 
		- C 구조체 헤더 정보 (약 40바이트)
		- p, y, t, h, o, n 6글자(6바이트) 
		- C-String 종단 문자 '\0'(1바이트)
		
		40바이트(헤더) + 6바이트(데이터) + 1바이트(Null) = 총 47바이트
		
	를 합쳐서 총 47바이트짜리 객체를 메모리에 생성합니다.
