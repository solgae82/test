1. 바이트 문자열 (Byte String)정의: 

	(관점): 0에서 255 사이의 정수(8비트)들이 연속되어 있는 나열을 의미합니다.

	(특징): 
	컴퓨터에서 텍스트(문자열)를 저장하기 위해 UTF-8이나 ASCII 같은 
	인코딩 규칙을 적용하여 문자를 이진 바이트로 변환한 형태입니다.
	
	(용도): 
	파일 I/O, 네트워크 통신 등에서 문자 데이터를 전송하고 저장하는 데 주로 사용됩니다.


2. 이진 시퀀스 (Binary Sequence)정의: 

	(관점): 0과 1로 이루어진 임의의 데이터 비트 또는 바이트 나열입니다.

	(특징): 
	이미지, 오디오, 실행 파일(.exe), 압축 파일 등 문자가 아닌 
	모든 형태의 이진 원시 데이터(Raw data)를 포함합니다.
	
	(용도): 텍스트뿐만 아니라 컴퓨터가 처리하는 모든 종류의 데이터를 아우르는 포괄적인 개념입니다

-------------------

바이트 문자열은 이진 시퀀스의 한 종류입니다.
이진 시퀀스가 의미론적으로 훨씬 더 큰 범주의 개념입니다

따라서 이진 시퀀스는 바이트 문자열일 수 있지만, 
이진 시퀀스가 반드시 문자(Text)만을 의미하지는 않기 때문에 
"모든 이진 시퀀스가 바이트 문자열인 것은 아닙니다".

-------------------

1. 껍데기가 없는 데이터 (물리적 관점)
	
	두 개념 모두 메모리에 올라가면 그저 0과 1로 구성된 연속된 숫자 배열일 뿐입니다.
	
	하드웨어 수준에서는 이진 시퀀스나 바이트 문자열이나 다를 바 없는 동일한 형태의 
	데이터 덩어리(Raw Data)입니다.
	
	"파이썬" 문자를 두 개념으로 만들었을때 이진수 나열 개념으로는 똑같습니다
		
		
2. 의미론적 차이 (해석의 관점), 타입으로 의미가 달라진다.

	(이진 시퀀스): 
	0과 1의 조합이 무엇을 뜻하는지 규정하지 않습니다. 
	이 데이터는 픽셀(이미지)일 수도 있고, 주파수(소리)일 수도 있으며, 
	CPU가 실행할 명령어(프로그램)일 수도 있습니다.

	(바이트 문자열): 
	이 0과 1의 조합이 "어떤 문자(Text)를 인코딩한 결과물"이라는 힌트를 내포하고 있습니다. 
	즉, 사람이 읽을 수 있는 글자로 디코딩(변환)될 준비가 되어 있는 이진 데이터입니다.

요약하자면, 컴퓨터 입장에서는 둘 다 똑같은 숫자 나열이지만, 프로그래머와 시스템 입장에서는 
"이건 문자열로 바꿀 수 있는 데이터야"라는 맥락(Context)을 명시한 것이 
바이트 문자열이라고 이해하시면 완벽합니다.

------------------- 코드로 확인

# 원본 문자열
text = "파이썬"

# 1. 바이트 문자열 (bytes) - 수정 불가능 (Immutable)
byte_string = text.encode('utf-8')

# 2. 이진 시퀀스 (bytearray) - 수정 가능 (Mutable)
binary_sequence = bytearray(text.encode('utf-8'))

# 결과 출력
print(f"1. 바이트 문자열 타입: {type(byte_string)}")
print(f"   출력 형태: {byte_string}\n")

print(f"2. 이진 시퀀스 타입: {type(binary_sequence)}")
print(f"   출력 형태: {binary_sequence}\n")



# 알맹이(이진 데이터)가 같은지 비교
print(f"두 데이터의 알맹이는 완전히 같을까? -> {list(byte_string) == list(binary_sequence)}")

# 출력
"""
1. 바이트 문자열 타입: <class 'bytes'>
   출력 형태: b'\xed\x8c\x8c\xec\x9d\xb4\xec\x8d\xac'

2. 이진 시퀀스 타입: <class 'bytearray'>
   출력 형태: bytearray(b'\xed\x8c\x8c\xec\x9d\xb4\xec\x8d\xac')

두 데이터의 알맹이는 완전히 같을까? -> True
"""
-------------------

bytes:(immutable)
	텍스트를 안전하게 저장하고 전송하기 위해 락(Lock)이 걸린 바이트 문자열상자입니다.

bytearray:(mutable)
	데이터 조작, 네트워크 버퍼링, 이미지 처리 등 
	이진수 자체를 마구 주물러야 할 때 사용하는 이진 시퀀스 상자입니다.

-------------------

위 코드를 통해 두 변수의 실제 이진수 데이터는 완벽히 같음을 확인할 수 있습니다.

알맹이는 동일하지만, 프로그래밍 언어가 제공하는 '타입(Type)'이라는 껍데기에 따라 
데이터를 바라보는 관점과 제약이 달라집니다. 

byte_string은 데이터를 변경할 수 없는 불변(Immutable) 상태이고, 
binary_sequence는 언제든 일부분 혹은 전체를 수정할 수 있는 가변(Mutable) 배열입니다. 
즉, 타입 시스템을 통해 동일한 데이터를 다루는 목적과 방식을 분리한 것입니다.

따라서 "두 변수의 값이 다르다"는 말은 물리적인 바이트 데이터가 다르다는 뜻이 아닙니다. 
언어가 그 데이터를 인식하고 다루는 내부적 규칙과 맥락(Context)이 다르다는 의미입니다.

개발자는 컴퓨터 내부의 복잡한 물리적 상태를 일일이 신경 쓸 필요가 없습니다. 
언어가 내부 장막(캡슐화) 뒤로 복잡한 메커니즘을 숨겨주고, 외부에 직관적인 API를 제공하기 때문입니다. 
개발자는 이 인간 친화적인 법칙과 사고방식(추상화)만 이해하고 프로그래밍하면 됩니다.

이것이 바로 파이썬 같은 고급 프로그래밍 언어가 인간에게 주는 가장 큰 혜택입니다. 
복잡함은 숨기고 본질에만 집중하게 만드는 것, 
모든 현대 프로그래밍 언어는 이 위대한 추상화의 원리 위에서 작동합니다.
