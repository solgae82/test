1. 필수 패키지 설치
	소스 컴파일을 위해 필요한 패키지를 설치합니다.

	dnf groupinstall "Development Tools" -y
	dnf install -y readline-devel zlib-devel openssl-devel wget

2. 소스 코드 다운로드 및 압축 해제 
	PostgreSQL 14 소스 코드를 다운로드합니다. 

	wget https://ftp.postgresql.org/pub/source/v14.12/postgresql-14.12.tar.gz
	tar -zxvf postgresql-14.12.tar.gz
	cd postgresql-14.12

3. 컴파일 및 설치

	./configure --prefix=/usr/local/postgresql-14.12
	make
	make install

4. 환경 설정 및 초기화

	# postgres 사용자 생성
	useradd postgres

	# 데이터 디렉토리 생성 및 권한 설정
	mkdir -p /usr/local/postgresql-14.12/data
	chown -R postgres:postgres /usr/local/postgresql-14.12
	
	# postgres 사용자 변경
	su -u postgres
	
	# 환경변수 설정	
	vi ~/.bash_profile
	..
	export PG_HOME=/usr/local/postgresql-14.12
	export PATH=$PG_HOME/bin:$PATH
	export PGDATA=$PG_HOME/data/svc01
	export PGLIB=$PG_HOME/lib
	:wq
	
	# 환경변수 적용
	source .bash_profile
	
5. 데이터베이스 클러스터 생성 

	# 데이터베이스 초기화	 
	initdb --locale='C' \
	--encoding='UTF8' \
	--pgdata='/usr/local/postgresql-14.12/data/svc01' \
	--waldir='/usr/local/postgresql-14.12/wal/svc01' \
	--auth='scram-sha-256' \
	--data-checksums \
	--pwprompt
	
	------ 
	# '--pwprompt' 설정으로 수퍼유저 암호를 설정하는 옵션이다
	
	initdb를 실행한 운영체제 (리눅스)사용자가 'postgres' 를 재사용해서  
	같은 계정 이름의 postgresql 계정 'postgres' 계정이 생성되고 이 계정이 superuser가 된다.
	수퍼유저 계정은 그렇게 정해지고, 그 다음 superuser 암호를 설정하게 된다.
	
	그래서 initdb 실행하는 운영체제 사용자를 먼저 만든 것이다.(postgres)

	위 initdb를 실행하면 아래와 같이 나올 것이다.
	------
	
	initdb .. #실행
	
	이 데이터베이스 시스템에서 만들어지는 파일들은 그 소유주가 "postgres" id로
	지정될 것입니다. 또한 이 사용자는 서버 프로세스의 소유주가 됩니다.

	데이터베이스 클러스터는 "C" 로케일으로 초기화될 것입니다.
	기본 텍스트 검색 구성이 "english"(으)로 설정됩니다.

6. postgresql 원격 설정
	
	cd $PGDATA #/usr/local/postgresql-14.12/data/svc01
	
	# postgresql.conf 설정 추가
	echo "listen_addresses='*'" >> postgresql.conf # 모든 원격 접속 가능하게 추가 설정
	
	# pg_hba.conf 설정 추가 
	vi pg_hba.conf
	
		# TYPE  DATABASE        USER            ADDRESS                 METHOD

		# "local" is for Unix domain socket connections only
		local   all             all                                     scram-sha-256
		# IPv4 local connections:
		host    all             all             127.0.0.1/32            scram-sha-256
		host    all             all             192.168.111.0/24        scram-sha-256 #추가
		host    all             all             192.168.0.0/24          scram-sha-256 #추가

	# 설정 파일 반영되게 리로드 
	
	cd $PG_HOME
	cd bin
	
	pg_ctl reload # 리로드 
	
7. 인스턴스 기동/정지
	#기동
	pg_ctl -D /usr/local/postgresql-14.12/data/svc01 -l logfile start
	
	#환경변수 PGDATA 를 설정했다면
	pg_ctl -l logfile start
	
	#기동 프로세스 확인 (-D 옵션을 붙여주면 찾기 편하다)

9. 서비스 등록 및 시작
	systemd 서비스 파일을 생성하여 등록합니다.
	
	vi /etc/systemd/system/postgresql-14.12.service
	파일 내용:
	
		[Unit]
		Description=PostgreSQL 14 database server
		After=network.target

		[Service]
		Type=forking
		User=postgres
		Group=postgres
		Environment=PGDATA=/usr/local/postgresql-14.12/data
		ExecStart=/usr/local/postgresql-14.12/bin/pg_ctl start -D ${PGDATA}
		ExecStop=/usr/local/postgresql-14.12/bin/pg_ctl stop -D ${PGDATA}
		Restart=always

		[Install]
		WantedBy=multi-user.target
		bash

	# 서비스 시작
	systemctl daemon-reload
	systemctl enable postgresql-14.12
	systemctl start postgresql-14.12
