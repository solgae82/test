
PostgreSQL에서 pg_ctl을 사용하여 데이터베이스 서버를 시작하거나 재시작할 때 
로그 파일의 위치와 동작을 설정하는 주요 옵션은 -l (log) 옵션입니다.

-------------------

주요 로그 옵션: -l (--log=FILENAME)

	이 옵션은 서버의 표준 출력(stdout)과 표준 에러(stderr)를 지정된 로그 파일로 리다이렉션합니다. 
	서버가 백그라운드에서 실행될 때 로그를 확인하는 데 필수적입니다.
	

로그 파일 위치 지정

	-l 옵션 뒤에 로그 파일의 전체 경로를 지정합니다.
		예: pg_ctl start -D /usr/local/postgres-14.12/data/lnxi \
		-l /usr/local/postgres-14.12/data/lnxi/logs/logfile.log

	
	#상대경로 일때 , 명령을 실행한 현재 위치에 파일 생성됨
		pwd
		/usr/local/postgres-14.12/bin
		
		pg_ctl start -l logfile.log # (이 파일 위치는?)
		
		#명령 실행한 현재 위치(/usr/local/postgres-14.12/bin) 에서 파일 생성
		ls
			..
			logfile.log 
	
	
로그 파일 덮어쓰기 vs 이어쓰기

	-l 옵션으로 지정된 파일은 매번 새로 생성되지 않고 기존 파일에 이어쓰기(Append) 됩니다.
	서버를 시작할 때마다 로그를 새로 만들고 싶다면, 
	start 명령 전에 기존 로그 파일을 삭제하거나 이름을 변경해야 합니다.

백그라운드 실행 (-w)
	
	-l 옵션과 함께 -w (wait) 옵션을 사용하면 서버가 완전히 시작될 때까지 pg_ctl 명령이 대기하므로, 
	시작 과정에서의 에러 로그를 즉시 확인할 수 있습니다.
	
	예: pg_ctl start -D ... -l ... -w

로그 파일 미지정 시
	
	-l 옵션을 사용하지 않으면, 로그가 터미널 화면에 출력되거나 백그라운드 실행 시 유실될 수 있습니다. 

PostgreSQL 설정 파일(postgresql.conf)을 통한 로그 관리

	pg_ctl -l은 서버 시작 시 stdout/stderr만 캡처합니다. 
	상세한 쿼리 로그, 에러 로그 레벨, 로그 파일 회전(rotation) 등은 
	postgresql.conf에서 설정해야 합니다.

	logging_collector = on
	log_directory = 'log'
	log_filename = 'postgresql-%Y-%m-%d_%H%M%S.log'
	
	# psql 쿼리로 로그 위치 확인
	show log_directory;
	
	# ps 로 -l 옵션 확인
	ps -ef | grep postgres
