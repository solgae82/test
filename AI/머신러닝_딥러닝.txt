예측한다는 것 = 함수를 찾는 것(규칙,패턴)

기계학습 = 데이터들 속에 패턴(함수)을 찾기 위한 과정

모델 = (학습해서 패턴을 찾은) 수학적 함수

AI = 모델 + 추론

AI는 가장 큰 범주이며, 그 안에 모델이 존재합니다.
	
	AI (인공지능): 인간의 지능을 모방하는 모든 기술
	
	Machine Learning (기계학습): 데이터를 통해 스스로 학습하는 AI의 한 방법론
	
	Model (모델): 기계학습을 통해 최종적으로 얻어낸 수학적 공식이나 판단 규칙(수학적 함수)

머신러닝(기계학습)의 학습의 의미
	
	데이터를 통해 특정 문제에 대한 해답을 내놓는 
	'함수(수식)'나 '규칙'을 컴퓨터가 스스로 찾아내게 하는 과정을 의미합니다.

	데이터에 맞는 최적의 가중치(weight)와 Bias를 찾는 것.
	
모델의 의미 

	머신러닝, 딥러닝을 통해 찾은 결과물(프로그램, 수학적 함수)
	
전통적 프로그램과 AI 모델 비교

	(전통적) 프로그램 : 코딩(패턴,규칙) -> 컴파일 -> 프로그램
	
	(AI 프로그램) 모델 : 대량데이터 -> 학습(패턴찾기) -> 모델(수학적 함수 프로그램)

머신러닝

	지도 학습 (Supervised Learning): 
	
		입력값(x)과 정답(Label,y)을 함께 학습하여 미래의 결과값을 예측 
		(예: 분류, 회귀)

	비지도 학습 (Unsupervised Learning): 
		정답 없이 데이터만으로 내재된 패턴이나 구조를 학습 
		(예: 군집화, 차원 축소)
	
	강화 학습 (Reinforcement Learning): 행동에 대한 보상을 통해 최적의 행동 방식을 학습. 
	
머신러닝 프로세스
	
	데이터 수집 및 전처리: 학습에 사용할 데이터를 수집하고, 결측치 처리, 정규화 등을 통해 모델이 학습하기 좋은 형태로 가공합니다.
	
	모델 학습 (Training): 전처리된 데이터를 머신러닝 알고리즘에 입력하여 데이터의 패턴과 규칙을 학습시킵니다.
	
	모델 평가 (Evaluation): 학습된 모델을 새로운 데이터(테스트 데이터)로 평가하여 예측 정확도를 검증합니다.
	
	모델 배포 (Deployment): 검증된 모델을 실제 서비스에 적용하여 예측 결과를 도출합니다. 

머신러닝 딥러닝 차이

	머신러닝(Machine Learning)은 데이터에서 규칙을 학습하는 인공지능 기술 
	
	딥러닝(Deep Learning)은 인간의 뇌 구조를 모방한 심층 신경망(Deep Neural Network)으로 
	대규모 데이터를 스스로 학습하는 머신러닝의 하위 분야입니다. 
	
	머신러닝은 수동적인 특성 추출이 필요하지만 
	딥러닝은 특징을 스스로 학습하는 것이 가장 큰 차이입니다. 
	
	핵심 차이점
		구조: 
			머신러닝은 통계적 알고리즘을 사용하며, 
			딥러닝은 다층 인공신경망(ANN)을 사용합니다.
		
		특징 추출: 
			머신러닝은 인간이 기계에 특징(Feature)을 정의해줘야 하지만, 
			딥러닝은 데이터에서 자동으로 특징을 추출합니다.
		
		데이터량: 
			딥러닝은 데이터가 많을수록 성능이 향상되지만, 
			머신러닝은 적은 데이터로도 학습이 가능합니다.
		
		컴퓨팅 자원: 
			딥러닝은 고성능 GPU 등 많은 컴퓨팅 자원이 필요하며, 
			머신러닝은 상대적으로 적은 자원이 필요합니다.
		
		해석 가능성: 
			머신러닝은 결과 도출 과정을 분석하기 비교적 쉽지만(White box), 
			딥러닝은 과정을 이해하기 어려운 '블랙박스'인 경우가 많습니다. 

학습과 추론의 차이

	학습(Training) : "지능을 만드는 과정"
		
		과정: 수많은 데이터를 컴퓨터에 밀어넣고, 모델이 스스로 정답과 오답을 비교하며 내부 수식(가중치)을 최적의 상태로 업데이트합니다.
		특징: 
			시간이 매우 오래 걸리고(며칠~몇 달), 전기와 비용이 많이 듭니다. 
			학습이 끝나야 비로소 하나의 '모델'이 완성됩니다.
	
	추론(Inference) : "지능을 사용하는 과정"
		
		과정: 이미 학습이 완료된 모델에 "이 사진 뭐야?"라고 물었을 때, 모델이 "이건 고양이입니다"라고 답을 출력하는 단계입니다.
		
		특징: 실시간으로 결과가 나와야 하므로 속도가 중요합니다. 
		우리가 스마트폰 앱에서 AI 기능을 쓰거나 챗GPT와 대화하는 모든 순간이 바로 추론 단계입니다.
	
	수만 장의 개 사진을 보고 '개'라는 특징을 배우는 것이 학습이고, 
	처음 보는 강아지 사진을 보고 "이건 개네요!"라고 알아맞히는 것이 추론
	
학습과 추론의 연산방식, 자원 차이

	1. 수학적 자원 (연산 방식)
	
		학습 (Training): "반복적인 미분과 수정"
			
			순전파(Forward) + 역전파(Backpropagation): 
				데이터를 넣고 결과값을 낸 뒤, 정답과의 오차를 계산하여 모델의 가중치를 거꾸로 거슬러 올라가며 수정합니다.
			
			미분(Gradient Descent): 
				오차를 최소화하기 위해 수만~수조 개의 파라미터를 미분하여 최적의 값을 찾습니다.
			
			고정밀도 연산: 
				미세한 오차 조정을 위해 보통 FP32(32비트 부동소수점) 같은 고정밀도 연산을 사용합니다.
			
		추론 (Inference): "단순한 행렬 곱셈"
			
			순전파(Forward) 전용: 
				이미 고정된 가중치에 입력값만 곱해서 결과값(확률)을 산출합니다. 
				역전파나 미분 과정이 없습니다.
			
			경량화(Quantization): 
				속도를 높이기 위해 FP16이나 INT8(8비트 정수) 등으로 연산 정밀도를 낮춰 효율을 극대화합니다.

	2. 물리적 장비 (하드웨어)
		
		학습 (Training): "거대한 창고와 슈퍼컴퓨터"
			
			GPU (H100, A100 등): 
				대규모 병렬 연산을 위해 수천 장의 고성능 GPU를 묶어서 사용합니다.
			
			고대역폭 메모리 (HBM): 
				방대한 데이터를 한꺼번에 읽고 쓰기 위해 매우 빠르고 비싼 메모리가 필수입니다.
			
			서버급 인프라: 
				엄청난 전력 소모와 발열을 감당할 수 있는 데이터 센터급 환경이 필요합니다.
		
		추론 (Inference): "민첩한 엔진"
			
			LPU / NPU: 
				추론에 최적화된 전용 칩(신경망 처리 장치)을 사용합니다. 
				최근 챗GPT 같은 서비스는 속도가 생명이라 추론 전용 가속기를 씁니다.
			
			에지 컴퓨팅 (Edge AI): 
				스마트폰, 자율주행차, CCTV 내부에 탑재된 작은 칩에서도 돌아가야 하므로 저전력, 소형화가 핵심입니다.
			
			CPU: 
				아주 가벼운 모델의 경우 일반 컴퓨터의 CPU만으로도 충분히 수행 가능합니다.
				
			메모리:  매우 많이 필요
				
추론에 메모리가 많이 필요한 주요 이유
	
	대규모 모델 적재: 
		학습된 생성형 AI 모델(LLM)의 크기가 거대해짐에 따라 이를 구동하기 위해 대용량의 RAM(DRAM)이 필요합니다.
	
	긴 문맥 처리 및 데이터 전송: 
		AI가 질문에 답하는 추론 과정에서 방대한 양의 텍스트 토큰을 생성해야 하므로, 
		메모리 내에서 쉴 새 없이 많은 데이터가 전송되는 병목 현상이 발생하여 이를 해결할 고속 메모리가 필수적입니다.
	
	실시간 서비스 요구: 
		AI 추론은 사용자에게 즉각적인 결과를 제공해야 하므로, 
		빠른 데이터 읽기/쓰기를 위해 고성능 GPU 메모리가 요구됩니다.
