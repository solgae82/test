처음엔 짜증날 것이다.
특히 IDE를 사용하면 IDE별로 구현 방식이 서로 달라서 짜증이 더 심하다.
이럴땐, 그냥 명령어 방식으로 연결하는 것이 제일 속편하다.
나중에 IDE의 GIT 사용법에 친해진 뒤에 연결을 시도하던지 하는게 낫다

---------

1. Github , dong(주인) 계정이 공유 리포지토리 생성 
	
	pj-1

2. Gihub dong계정이 pj-1 플젝에 커밋할 사용자 추가
	settings > Manage access > Manage access > 텍스트 박스
	: 사용자(ID) 검색 후 추가
	추가된 사용자에게 확인 메일이 발송된다.
	
3. 추가된 사용자는 본인 깃허브에 등록된 메일 확인해서 verify 해야한다.
	- 메일에서 verify 해야 dong계정의 pj-1에 커밋 가능.

4. 인증된 사용자는 본인 토큰이 필요하다
	- 현재 GitHub에는 내 비밀번호로 커밋하는게 아니고, 
	토큰을 생성해서 커밋한다. 
	(깃허브_토큰로그인변경.txt) 참조
	
4. IDE를 이용하든, 명령어를 이용하던 첫 커밋이 중요!
	- 새로 생성한 리포지토리에 아무 파일이 없다면 push로 첫 커밋 반영 가능
		- 새로 생성한 곳에 미리 생성된 파일이 있다면(readme 파일 등)
		pull, 또는 fetch 병합 후 첫 push 시도해야함
		
	- 첫 연결 배포자의 파일을 나머지 연결 사용자가 받고 나서 시작하면 편함.
	
	- 서로 각자 따로 작업한 것 push 하려면, 언제나 pull 후 해야함.
