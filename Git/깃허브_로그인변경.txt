2021.8.13일자로 
기존 아이디/비밀번호로 접속해서 git push / pull 하던 방식을,
아이디/(토큰) 방식으로 바꿨다고 함.

접속하려는 아이디로 github사이트 로그인 후 
우측 메뉴에서 개인 프로필 이미지 클릭 > 
Settings > Developer settings > Personal access tokens > Tokens(classic)

Generate new token 누르고 토큰 생성하는데, 
	Note : 토큰명 (이 문자로 토큰을 생성한다..'아이디_token' 등으로 생성)
	expire : 토큰 사용 기간을 설정하는데 'No expiration'은 기간 없음이다.
	Select Scopes :
		repo , admin:org , admin:repo_hook등을 체크한다 (repo만 선택하고 해도 된다)
		(아마도 github 사이트 서비스들 같은데, 자세한 것은 잘 모르니 대충 짐작으로 repo는 체크해야할 것이다)

마지막 Generate token 버튼으로 생성하는데,	
다음 페이지에서 처음 한번만 생성된 토큰 문자열이 보인다
잘 Copy해서 보관해야한다..(다시 확인 불가능)

또한 그 계정의 이메일 주소로도 토근을 생성했네 하는 메일도 올 것이다..
그 메일에서 인증도 해야한다..

깃허브 연결하려는 여러 툴들로 연결할때, 연결하려는 아이디 입력 후 기존 로그인 비밀번호 입력이 아닌, 
방금 생성한 토큰 문자열로 대체해서 연결하면 연결될 것이다.

윈도우 재설치 후 기존에 있던 git 디렉토리를 연결하다가 알게된 점.

토큰방식으로 로그인을 하면 깃에서 설정하던 user.name , user.email 등의 config 설정을 쓰지 않는다
MS 윈도우라면 [windows 자격 증명] 이라는 곳에 설정하고 그곳에 설정된 자격증명 정보로 깃허브에 로그인 하게 된다

	제어판 > 자격증명 관리자 > Windows 자격 증명 > 일반 자격 증명 
	: 이곳에 내가 github.com 에 git 연결한 정보들이 있다.
	
한번 깃허브와 연결된 후 --global , --local 등의 config 의 user.name , user.email 등을 바꿔보고 push등을 
해봤는데, 계속 이전 연결 아이디로만 push를 하길래? 이게 뭐야 하고 검색 등을 해보다가 이런 사실을 알게 되었다.

윈도우 재설치등을 하면 윈도우에 시스템에 저장한 자격증명은 날라갔을 것이므로, 
	초기화 모드로 다시 로그인: 
		깃허브 로그인 화면 뜨고 > 토근 모드 설정 후 토큰 입력하면 > 그 토큰 관련 ID / password를 물어보게 된다
