이클립스에서 증분 빌드(Incremental Build)는 프로젝트의 변경된 부분만 효율적으로 다시 빌드하는 기능입니다. 
전체 코드를 처음부터 다시 빌드(Clean Build 또는 Full Build)하는 대신, 
마지막 빌드 이후에 변경되거나 새로 추가된 소스 코드 파일만 컴파일하고 처리하여 빌드 시간을 크게 단축시킵니다. 

주요 특징 및 작동 방식
	
	효율성: 대규모 프로젝트에서 전체 빌드는 시간이 오래 걸릴 수 있는데, 
		증분 빌드는 변경된 파일과 그에 영향을 받는 파일만 처리하므로 개발 생산성이 향상됩니다.
	
	자동화: 이클립스는 기본적으로 "Build Automatically" 옵션(Project 메뉴에서 설정 가능)을 제공하며, 
		이 옵션이 활성화된 상태에서 소스 파일을 저장하면 자동으로 증분 빌드가 트리거됩니다.
	
	컴파일: 주로 Java 소스 파일(*.java)이 수정되면 해당 파일만 다시 컴파일하여 바이트 코드(.class 파일)를 업데이트합니다.
	
	빌드 상태: 이클립스 내부적으로 이전 빌드 상태를 추적하고, 
		리소스 변경 델타(resource change delta)를 기반으로 최적화된 빌드를 수행합니다. 
	
요약하면, 증분 빌드는 개발 과정에서 실시간으로 빠르게 변경 사항을 반영하고 확인할 수 있도록 돕는 이클립스의 핵심 기능 중 하나입니다.
