	

chage 옵션 계정명

옵션 
-d : 1970 1월 1일 부터 패스워드를 마지막으로 변경한 일자(/etc/passwd에 해석으로)(설정은 YYYY-MM-DD로 한다) 
-E : 계정사용 종료일자 (YYYY-MM-DD) 
-I : 패스워드가 유효기간 종료 이후에 계정 비활성화될 일 수 
-m : 패스워드를 변경할 수 있는 최소일 수 (0 이면 아무때나 변경가능) 
-M : 마지막 변경 이후의 패스워드 유효 일 수 (99999 이면 무한정)
-W : 패스워드 만료기간 몇 일전에 안내메시지를 보낼 것인가. 
-l(소문자 L) : 특정사용자의 정보출력 


chage -d 2015-11-22 dong
chage -E 2016-12-31 dong
chage -I 30 dong
chage -m 10 dong
chage -M 90 dong (99999로 지정하면 무한정 쓸 수 있다)
chage -W 7 dong


chage -l dong

Last pasword chage(마지막으로 암호 바꾼 날)					: Nov 22, 2015
Password expires(암호 만료일)							: Feb 20, 2016			
Password Inactive(암호비활성화기간)						: Mar 21, 2016
Account expires(계정만료일)							: Dec 31, 2016
Minimum number of days between password change(암호를 바꿀 수 있는 최소 날 수)	: 10
Maximum number of days between password change(암호를 바꿔야 하는 최대 날 수)	: 90
Number of days of warning between password expires(암호 만료 예고를 하는 날 수)	: 7


----------
암호 만료일 연산은: 
Password expires(암호 만료일) = Last pasword chage + Maximum number of days between password change
그러므로 위 예제의 암호만료일은 Nov 22, 2015 + 90일 = Feb 20, 2016 일이된다.

암호 만료일이 결정되어야 나머지 연산들이 가능해진다. 그러므로 다른 옵션들도 이것이 지정되어지지 않는다면
소용없게 된다.

---------
암호비활성화 기간 =  (암호만료일 기준 + 암호비활성화일)

---------
chage -E 2016-12-31 dong
usermod -e 2016-12-31 dong 
위 둘은 같다

---------
계정락풀기
password -l dong <- 2단계 락 걸기
password -u dong <- 2단계 락 풀기

usermod -L dong <- 1단계 락 걸기
usermod -U dong <- 1단계 락 풀기

password -l dong <- 2단계 락 걸기
usermode -U dong
usermode -U dong <- 두번해서 락 풀기
