
구글 제미나이 정리: 

도커 볼륨(Volume) 마운트와 바인드 마운트(Bind Mount)는 컨테이너 데이터를 영구 저장하는 방식입니다. 

볼륨 마운트는 도커가 관리하는 저장 영역으로 데이터 보존 및 공유에 적합하며, 
바인드 마운트는 호스트의 특정 폴더를 연결하여 소스 코드 개발 등 실시간 수정이 필요할 때 주로 사용합니다. 

주요 차이점:
	관리 주체: 볼륨은 도커가 관리(var/lib/docker/volumes/ 등)하여 안전하고, 
		바인드 마운트는 사용자가 호스트의 임의 경로를 직접 지정합니다.
	
	보안 및 성능: 볼륨은 도커 엔진이 관리하므로 격리되어 안전하고 성능이 좋습니다. 
		반면, 바인드 마운트는 호스트 파일 시스템에 직접 접근하므로 보안상 주의가 필요합니다.
	
	영구 저장: 컨테이너가 삭제되어도 볼륨 데이터와 바인드 마운트(호스트 파일/디렉토리)는 남아있으나, 
		바인드 마운트는 연결된 호스트 파일/디렉토리가 삭제되면(사용자 실수로) 함께 사라질 수 있습니다.
		(볼륨은 도커가 관리하는 영역이라 안전하다는 뜻)

활용 사례:
	볼륨: 데이터베이스, 지속적인 데이터 저장, 운영 환경 (Production).
	바인드 마운트: 소스 코드 실시간 반영, 설정 파일 공유, 개발 환경 (Development). 

볼륨은 관리 용이성, 바인드 마운트는 편리한 파일 공유에 강점이 있습니다

주로 리눅스에서 사용할 것이므로 volume 생성시 도커 관리 디렉토리인 아래 경로를 기억하자
	
	var/lib/docker/volumes/
	
--------- volume 생성 및 관리

# 볼륨 호스트에 생성
$ docker volume create my-data

# 생성된 볼륨 조회
$ docker volume ls
DRIVER    VOLUME NAME
local     my-data


#생성된 볼륨 정보 보기
$ docker volume inspect my-data
[
    {
        "CreatedAt": "2026-02-24T22:58:53Z",
        "Driver": "local",
        "Labels": null,
        "Mountpoint": "/var/lib/docker/volumes/my-data/_data",
        "Name": "my-data",
        "Options": null,
        "Scope": "local"
    }
]

#생성한 my-data 컨테이너에서 사용
	# run 명령시 -v 옵션으로 설정
	- v옵션 : (호스트 디렉토리):(컨테이너 안 앱 저장디렉토리)
	
	$ docker run --name db -v my-data:/var/mysql mysql

--------- 바인드 마운트

직접 호스트의 특정 위치로 영구데이터 관리하고 싶을때 
만약 컨테이너에 데이터가 이미 쌓여있는데, 비어 있는 호스트 디렉토리를 run -v 옵션으로 연결한다면,
호스트 기준으로 덮어써지므로 주의!!

	# run 명령시 -v 옵션으로 설정
	- v옵션 : (호스트 디렉토리):(컨테이너 안 앱 저장디렉토리)
	
	$ docker run --name db -v C:\mysql:/var/mysql mysql
