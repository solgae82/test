 컨테이너 내부 볼륨과 호스트 볼륨 동기화시 제외할 디렉토리나 파일이 있다면,
 빈 디렉토리/파일을 덮어 쓰는 방식으로 해결한다
 
 예를 들어 파이썬 앱의 가상환경 디렉토리의 공유를 막아야 한다면 다음과 같이 설정한다.
 
 run 설정시

	 docker run -d \
	  -p 5000:5000 \
	  -v .:/app \
	  -v /app/venv \
	  --name flask-dev \
	  my-flask-app
	  
  yaml 설정시
  
	  volumes:
	  - .:/app          # 1번: /app 전체를 호스트와 연결 (venv까지 통째로 들어옴)
	  - /app/venv       # 2번: /app/venv 경로만 "빈 공간"으로 덮어써서 가려버림
	  
설명 :

	(호스트디렉토리:컨테이너디렉토리)
	.:/app <= 호스트 현재디렉토리 와  컨테이너 /app 디렉토리 동기화
	
	 - .:/app 실행 시점컨테이너의 /app 폴더가 호스트 폴더와 똑같이 변합니다. 
	 이 과정에서 호스트에 있던 잘못된 venv 폴더도 컨테이너의 /app/venv 자리에 그대로 들어옵니다.
	 
	 - /app/venv 실행 시점도커가 텅 비어있는 깨끗한 빈 폴더를 가져다가 컨테이너의 /app/venv 자리에 
	 스티커처럼 위에 덧붙여 덮어씌워 버립니다.
