도커의 진정한 강력함인 "한 번만 빌드하고, 어디서나 실행한다 
(Build Once, Run Anywhere)"는 개념이 있다

그러나 실제론 CPU 아키텍쳐에 따라서 결과물이 달라진다.

크게 보면 제작환경 CPU 와 실행될 곳 CPU가 같은 종류냐, 아니냐로 구분된다
같은 종류 아키텍쳐 CPU라면 그냥 pc에서 이미지 빌드하고 서버에 이미지 설치하면 된다.

다른 종류 아키텍쳐라면 실행될 곳에 맞는 옵션을 주고 빌드한다.
또는 그 실행환경에 가서 직접 빌드한다.

------------------- 이미지 (윈도우 도커 빌드 -> 리눅스 도커) 배포

CPU 아키텍쳐가 같다면 윈도우 도커에 빌드한 이미지도 리눅스 도커에서 실행된다.
(CPU 아키텍쳐 다르다면 옵션을 줘서 이미지 빌드해야한다.)

이미지 배포는 크게 3가지 방법이 있다
(CPU 아키텍쳐가 서로 맞다고 가정)

	1. PC 로 이미지 빌드 > Doucker Hub에 배포  
		> 사용할 환경(리눅스)에서 pull로 다운 받아서 사용.
		
	2. 폐쇄망일 경우 
		
		# PC (이미지 빌드): 
		docker save -o flask-app.tar [내_이미지_이름]
		
		# flask-app.tar > 리눅스에 업로드 후
		docker load -i flask-app.tar
		
	3. 실핼될 리눅스에서 직접 (Dockerfile 등을 이용) 도커 이미지 빌드
	
	
------------------- 이미지 (윈도우 도커 -> 리눅스 도커, CPU 아키텍쳐 다를때) 배포 개념

	1. CPU가 다른 PC 환경과 리눅스 환경 맞게 이미지 배포하는 방법
	
		- Docker Hub 이용 방식
			- 윈도우 도커 데스크탑엔 애뮬레이션(QEMU)이 내장되어서, 
			다른 CPU 아키텍쳐로 빌드해도 돌아간다(pc에서 성능은 저하)
			
			- Multi-Platform 빌드 방식을 이용하면 두 환경 모두 만족하는 이미지 빌드 가능			
			
		- 폐쇄망
			 Multi-Platform 빌드 방식으로 특정 환경으로 이미지 빌드
	
	2. 리눅스 도커에서 직접 이미지 빌드
		폐쇄망일 경우에는 리눅스 도커에서 직접 이미지 빌드도 대안적인 방법.

------------------- 이미지 (윈도우 도커 -> 리눅스 도커, CPU 아키텍쳐 다를때) 빌드 배포 실전 코드

PC 와 리눅스 CPU  환경이 다를때
(pc: amd64 , 리눅스: arm64)

1. Docker Hub 배포

	# 두 가지 CPU 구조(인텔용, 맥북/ARM용)를 한 번에 빌드해서 도커에 배포하는 명령어
	docker buildx build --platform linux/amd64,linux/arm64 -t 내계정/flask-app:1.0 --push .

2. 폐쇄망 배포
	--load 옵션: 이미지 생성 후 로컬에 저장
	
	# 내 컴퓨터(윈도우)에서 실행하지만, 리눅스 서버용(arm64)으로 구워서 로컬에 저장하는 명령어
	docker buildx build --platform linux/arm64 -t my-flask-app:linux-version --load .
	
	# 리눅스 폐쇄망 배포용 .tar 만들기
	docker save -o flask-app.tar my-flask-app:linux-version

	# 리눅스 (업로드 후) 도커 배포:
	docker load -i flask-app.tar
	
	# 이미지 설치 확인
	docker images
