도커이미지 + 구성 할때 아래 두가지 파일이 함께 쓰인다

	Dockerfile: 
		우리 회사/개인의 커스텀 애플리케이션 이미지를 만드는 데 사용.

	docker-compose.yml: 
		커스텀 이미지와 기성 이미지(MySQL 등)를 
		조립하여 최종 서비스를 구동하는 데 사용.
		

Dockerfile은 이미지 빌드할때 사용된다.
앱이 돌아갈 기본환경 셋팅등를 여기서 해 놓는다

	FROM python:3.11			# 1. 기본 이미지 지정
	WORKDIR /app				# 2. 작업 디렉토리 설정
	COPY . .				# 3. 내 소스코드 전부 복사
	RUN pip install -r requirements.txt	# 4. 패키지 설치 (필요시)
	CMD ["python", "main.py"]		# 5. 실행할 명령어
	
docker-compose.yml은 애플리케이션
환경 변수 설정, 포트 개방, 데이터베이스 연결, 컨테이너 실행 명령어(CMD) 변경 등을 위한 레이어로 이해하자
쉽게 말하면 실제 앱이 돌아가야할때 필요한 모든 것을 묶는 작업.

docker commit 명령어로 이미지 만들고 docker-compose 로 구성 해도 되겠으나, 
Dockerfile로 미리 셋팅하는 이유가 있다.

	- 이미지가 어떻게 설정됐는지 이력이 없다(그때 기억에 의존하거나, 따로 문서를 작성해야함)
	- 용량이 커진다(수동 셋팅 과정에 생긴 캐시파일등이 이미지에 포함됨)
	
이런 쓰임새로 Dockerfile과 함께 쓰이므로, 기본 사용법을 같이 알아둬야한다.
실전에서는 docker-compose 실행할때 Dockfile 이미지 빌드 포함해서 구성하는 전략을 취한다.
따로 분리해서 사용해도 무관하나, 같이 연동하는 방식이 가장 최적화된 기법.

이런 이유로 GitHub등의 오픈 소스 앱 디렉토리를 보면 
Dockerfile, docker-compose.yml이 함게 있는 것이다.
