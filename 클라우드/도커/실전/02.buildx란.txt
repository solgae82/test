------------------- buildx
<멀티 플랫폼 빌드를 위한 차세대 확장 빌드 도구>

buildx는 도커가 제공하는 "차세대 확장 빌드 도구"입니다.
멀티 플랫폼 빌드를 해도 우리 눈에는 여러 개의 이미지가 아니라, 
동일한 이름과 태그를 가진 '딱 한 개의 이미지'로 보입니다.

내계정/flask-app:1.0 (대표 이름)
├── 📦 인텔/AMD용 실제 이미지 (linux/amd64)
└── 📦 맥북/ARM용 실제 이미지 (linux/arm64)

도커 엔진이 다운로드받기 직전에 "어? 이 리눅스 서버는 x86_64(인텔) 구조네? 
그러면 저 폴더 안에서 인텔용 알맹이만 쏙 빼서 다운로드해야지!"라고 알아서 판단합니다

자동 선택 기능 다운로드 기능은 오직 Docker Hub 사용 구조에서만 작동합니다.

------------------- 실전

# (윈도우/intel)에서 두 가지 CPU 구조(인텔용, 맥북/ARM용)를 Multi-Platform 빌드해서 도커에 배포하기

	docker buildx build --platform linux/amd64,linux/arm64 -t 내계정/flask-app:1.0 --push .

# (윈도우/intel)에서 (리눅스/arm64)용으로 빌드해서 도커에 배포하기

	docker buildx build --platform linux/arm64 -t 내계정/flask-app:1.0 --push .

# (윈도우/intel)에서 (리눅스/arm64)으로 로컬 이미지로 생성 저장하는 명령어
# 빌드한 이미지 파일은 윈도우 도커에서는 안 돌아간다(당연)
	
	docker buildx build --platform linux/arm64 -t my-flask-app:linux-version --load .

	# 리눅스 업로드용 파일 만들기
	docker save -o flask-app.tar my-flask-app:linux-version 

	# 리눅스에 파일 업로드 후 설치
	docker load -i flask-app.tar
