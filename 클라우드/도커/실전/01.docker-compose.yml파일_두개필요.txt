
docker-compose.yml 은 이미지 빌드할때만 필요한게 아니라, 실행할 때도 필요하다.
빌드할때와 실행할때는 옵션이 달라질 수 밖에 없다.
그러므로 두개가 필요하다.

	docker-compose-build.yml (이미지 빌드용)
	docker-compose.yml (컨테이너 실행용)

항상, docker compose 명령어 실행할때 위 파일이 있는 곳에서 해야한다.
docker compose 명령은 파일 지정이 되지 않으면 현재 디렉토리의 
	docker-compose.yml 를 기준으로 실행된다.

이미지 빌드할때는 아래와 같이 한다(이미지 빌드)
	
	docker compose -f docker-compose.build.yml build
	
	#도커허브에 올리기
	docker compose -f docker-compose.build.yml push 

컨테이너 최초 실행 생성 할때는 아래와 같이 한다
	(현재 디렉토리의 'docker-compose.yml' 을 실행)

	# 컨테이너 새로 생성, 백그라운드 실행 -d
	docker compose up -d
	
	# 컨테이너 삭제 
	docker compose down
	
생성된 이후부터는

	docker compose stop	# 컨테이너 중지
	docker compose start	# 컨테이너 시작(항상 백그라운드 실행)
	docker compose restart  # 컨테이너 재시작
	
이런 식이므로, 컨테이너 명령어 stop/start를 쓸일이 없다.

docker compose 가 컨테이너 실행/중지/삭제 기준 명령어가 된다.


-------------------

도커는 나중에 정의한 규칙이 이전에 정의한 규칙을 덮어쓴다
그래서 docker-compose-build.yml 로 이미지 빌드(기본버전)하고, 
다시 docker-compose.yml 로 환경변수를 덮어써서 운영버전으로 컨테이너를 만든다
