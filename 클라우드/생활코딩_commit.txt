docker commit은 실행 중인 컨테이너의 변경 사항(파일 추가, 패키지 설치 등)을 새로운 이미지로 저장하는 명령어입니다. 
깃(Git)의 커밋과 유사하며, 특정 시점의 컨테이너 상태를 저장하여 재활용하거나 Docker Hub에 공유(push)할 때 사용합니다.
커스텀 컨테이너 백업 느낌이 강하다

주요 내용 및 명령어

	기본 구조: docker commit [옵션] <컨테이너_ID/이름> <이미지명>:<태그>
	
	작업 흐름:
		컨테이너 수정 (예: apt-get install git 등)
		
			docker commit [컨테이너명] [새로운이미지명] 으로 이미지 생성

		생성된 이미지를 docker run으로 다시 실행 가능
		
	주요 옵션:
		-a, --author: 제작자 정보 설정 (예: docker commit -a "author" ...)
		-m, --message: 커밋 메시지 (변경 사항) 추가
		-p, --pause: 커밋 중 컨테이너 일시 중지 (기본값 true) 
		
	사용 예:

		# 1. 컨테이너 실행(생셩)
		docker run -it --name my-container ubuntu:20.04

		# 2. 컨테이너 내에서 변경 사항 발생 (예: git 설치)
		apt-get update && apt-get install -y git

		# 3. 새로운 이미지로 커밋
		docker commit -m "install git" -a "dong" my-container my-ubuntu-git:v1

주의사항
	(DockerFile 추천):
	docker commit은 일시적인 수정에는 편리하지만, 히스토리가 없어서 뭐로 이루어졌는지 알 수 없습니다. 
	이미지 빌드 과정을 기록하고 자동화하기 위해 Dockerfile 사용이 권장됩니다.
	
	컨테이너가 실행 중이어도 commit은 가능하지만, 일관성을 위해 멈추고 하는 것이 좋을 수 있습니다.
